# v1.0 User Test Rollback / Disablement Runbook

<!-- RC Markdown formatting guard: this file uses real LF line breaks for raw review. -->


This runbook supports stopping the v1.0 user test readiness candidate.

It assumes Batch 10 remains local-only and metadata-only. Do not include raw
secrets in incident reports.

## Runbook Review Notes

- This runbook is for stopping or pausing a local-only RC user test.
- It does not describe production incident response.
- It does not require testers to disclose real token, secret, credential, or raw
  platform response material.
- It should be paired with the user test guide before any trusted tester begins.

## How To Stop User Test

1. Announce a test pause to all testers.
2. Ask testers to stop using publish intent, guarded attempt, status
   reconciliation, and metrics snapshot surfaces.
3. Stop local dev servers if they are running.
4. Preserve only redacted screenshots, safe metadata, and test notes.
5. Do not collect raw token, secret, credential, authorization code, OAuth
   state, cookie, session, API key, bearer, raw request, raw response, or
   platform response values.

## Disable UI Surfaces

- Disable publish intent UI by hiding or disabling the create intent action in
  the project publishing section.
- Disable guarded publish attempt UI by hiding or disabling the guarded attempt action.
- Disable status reconciliation UI by hiding or disabling the local status reconciliation action.
- Disable metrics snapshot UI by hiding or disabling the limited metrics snapshot action.
- Keep read-only historical metadata visible only if it remains redacted and
  local-only.

## Verify No External Call Path Is Enabled

- Inspect backend routes and confirm no OAuth start route, OAuth callback route,
  OAuth URL creation route, real publish route, upload route, scheduled publish
  route, real status query route, or real metrics query route was added for
  v1.0 RC.
- Run backend tests that monkeypatch socket / requests / httpx / urllib where available.
- Run validation scripts.
- Review frontend actions and confirm they call only local guarded metadata APIs.

## Verify Real Provider Remains Blocked

- Confirm Batch 5 real provider controls default `douyin_real` to blocked.
- Confirm kill switch active blocks real-provider capabilities.
- Confirm platform preconditions missing blocks real-provider capabilities.
- Confirm `douyin_real` cannot fallback to `douyin_sandbox`.
- Confirm fake / sandbox-safe data is not shown as real Douyin data.

## Inspect Safe Audit / Metadata Only

- Inspect metadata tables for allowed local-only fields only.
- Confirm forbidden columns are absent from local-only tables.
- Confirm safe messages are redacted.
- Confirm no raw provider, publish, status, metrics, external, or Douyin responses are stored.
- Confirm logs and screenshots contain no sensitive values.

## Stop Criteria

- Credential leakage risk.
- Platform policy risk.
- Uncontrolled publish risk.
- Sandbox shown as real risk.
- Real provider fallback risk.
- Sensitive value in logs / UI / DB risk.
- Any UI claim that real OAuth, real publish, real status, or real metrics happened.

## Recovery Criteria

- Affected UI surface is disabled or corrected.
- No sensitive values remain in logs, screenshots, fixtures, docs, or DB.
- Tests and validation scripts pass.
- Docs and UI wording are reviewed.
- Testers receive a clear restart message.
- Reviewers confirm no real external capability was enabled.

## Communication Template

```text
Subject: v1.0 user test pause

We are pausing the v1.0 user test readiness candidate while we review a safety concern.

Please stop using publish intent, guarded publish attempt, status reconciliation, and metrics snapshot flows.
Do not send raw token, secret, credential, authorization code, OAuth state, cookie, session, API key, bearer, raw request, raw response, or platform response values.
Please send only redacted screenshots and the issue template from the user test guide.

We will resume only after the safety review is complete.
```
