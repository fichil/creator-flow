# ADR 0004: 定时生成待审核草稿

## 状态

Accepted

## 背景

creator-flow 的原始目标包含可持续的内容运营能力。用户需要配置账号定位、内容类型和生成频率，让系统按计划从近期显式导入素材中生成候选视频草稿。

同时，平台发布和用户控制之间必须有明确边界。自动化可以提升持续产出能力，但不能替代用户对内容表达、隐私风险、版权风险和发布动作的判断。

## 决策

允许定时自动生成待审核草稿，但禁止定时自动发布。

`Scheduler / Trigger Engine` 只触发草稿生成。它可以创建 `GenerationRun`，基于用户显式导入的近期素材和用户明确启用的热点来源生成候选草稿，并将结果放入 `Review Queue`。

任何发布、排期发布或上传用于公开发布的动作，都必须经过 `Review Queue` 与用户明确确认。

## 结果

- 用户可以配置内容生成频率，支持更稳定的内容运营节奏。
- 自动生成结果默认处于待审核状态。
- `Scheduler`、`Review Queue` 和 `PublisherProvider` 的职责边界清晰。
- 热点来源必须可配置、可关闭，并记录来源、采集时间和用户启用状态。
- 系统不能因为存在计划任务而绕过人工确认发布原则。
