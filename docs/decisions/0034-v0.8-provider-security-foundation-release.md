# ADR 0034: v0.8.0 Provider & Credential Security Foundation Release

## Status

Accepted

## Context

v0.7.0 completed the local fake/manual metrics review summary workflow.

v0.8 Batch 1 through Batch 16 established and audited the Provider & Credential Security Foundation:

- Provider / Credential / OAuth / Secret / token lifecycle documentation boundaries.
- Provider Registry & Capability Metadata.
- Provider Connection State.
- Provider Credential Reference & Secret Redaction.
- Provider Security Audit Event & Redacted Audit Log.
- Provider OAuth State & Callback Boundary.
- Provider Token Lifecycle Boundary.
- Provider Integration Readiness Summary.
- Matching frontend read-only UI panels.
- RC checklist and closure audit.

The Batch 16 RC audit confirmed that v0.8 remained metadata-only / read-only, did not add real Douyin integration, did not implement OAuth, did not store tokens or credentials, did not fetch real metrics, and did not upload, publish, schedule, or call external services.

The project now needs a release decision that promotes this audited foundation from RC to stable release while preserving the same security and source-separation boundaries.

## Decision

v0.8.0 is released as Provider & Credential Security Foundation.

v0.8.0 promotes the v0.8 Batch 1-16 security boundary foundation from RC to stable release.

v0.8.0 updates the current stable release from `v0.7.0` to `v0.8.0 - Provider & Credential Security Foundation`.

v0.8.0 does not include real Douyin integration.

v0.8.0 does not include OAuth implementation.

v0.8.0 does not include OAuth callback routes, OAuth state storage, token exchange, or real provider authorization URL generation.

v0.8.0 does not include token storage.

v0.8.0 does not include real Credential storage.

v0.8.0 does not include a real provider adapter.

v0.8.0 does not fetch real metrics, upload, publish, schedule, or call external services.

## Release Scope

v0.8.0 includes:

- Documentation and ADR boundaries for Provider, Credential, OAuth, Secret, token lifecycle, security audit, connection state, and source separation.
- Backend read-only metadata APIs for Provider Registry, Connection State, Credential Reference, Security Audit, OAuth Boundary, Token Lifecycle Boundary, and Integration Readiness Summary.
- Metadata-only tables for provider connection state, credential reference, security audit events, OAuth boundaries, and token lifecycle boundaries.
- Backend secret redaction helper.
- Frontend read-only UI panels for Provider Registry, Connection State, Credential Reference, Security Audit, OAuth Boundary, Token Lifecycle Boundary, and Integration Readiness Summary.
- RC checklist / closure audit.
- Release notes and annotated `v0.8.0` tag.

## Security Boundary

v0.8.0 preserves `fake_local`, `douyin_sandbox`, and `douyin_real` source separation.

`fake_local` is local fake/demo/test workflow only and must not be interpreted as real Douyin readiness.

`douyin_sandbox` is sandbox placeholder metadata only and must not be treated as `douyin_real`.

`douyin_real` is future real provider placeholder metadata only and must not be interpreted as connected, authorized, token stored, metrics ready, publishing ready, v0.9 POC complete, or production ready.

API responses and frontend panels may show non-sensitive metadata, source type, implementation status, policy/status fields, safe messages, boundary notes, readiness items, blocking reasons, and next safe steps. They must not expose access tokens, refresh tokens, token values, API keys, secrets, client secrets, authorization codes, OAuth client secrets, OAuth state values, credential material, private keys, raw requests, raw responses, raw payloads, token expiry values, token refresh responses, token revoke responses, provider token responses, or real provider responses.

## Consequences

The current stable release becomes v0.8.0.

Users and reviewers can inspect provider/security/readiness boundaries without being offered real platform actions.

Future work can build on the provider metadata, source separation, redaction helper, OAuth boundary metadata, token lifecycle boundary metadata, and readiness summary, but must not treat the v0.8.0 metadata foundation as real provider integration.

v0.8.0 is not production readiness certification. It is a foundation release that makes the security and source boundaries reviewable before any Douyin POC begins.

## v0.9 Entry Boundary

v0.9 may start Douyin Provider POC / Sandbox Integration only after this release.

v0.9 must start with a separate ADR, separate branch, tests, and security-boundary review.

v0.9 must preserve `fake_local`, `douyin_sandbox`, and `douyin_real` separation.

v0.9 must not silently turn placeholders into real provider behavior.

v0.9 must not store real tokens until Credential storage is separately designed and reviewed.

v0.9 must not add real OAuth, real token lifecycle operations, real metrics fetching, upload, publish, scheduling, or external service calls without explicit scope, tests, and security review.
