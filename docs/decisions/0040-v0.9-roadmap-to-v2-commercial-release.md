# ADR 0040: v0.9 Roadmap to v2.0 Commercial Release Alignment

Status: Accepted

## Context

v0.8.0 released the Provider & Credential Security Foundation. v0.9 Batch 0 through Batch 4 then established the Douyin Provider POC planning, backend-only adapter skeleton, sandbox-only deterministic operation simulation, provider registry / factory routing, and sandbox metrics / mock workflow POC.

The project now needs an explicit roadmap from the current v0.9 POC stage through v1.0 user testing, v1.5 minimum production readiness, and v2.0 multi-tenant SaaS commercialization. The roadmap must distinguish current capabilities from future readiness targets so that sandbox POC work is not mistaken for production, commercial, or SaaS readiness.

## Decision

v0.9 Batch 5 is docs-only / planning-only. It adds roadmap and readiness documentation for:

- v1.0 Douyin Integration User Test Release.
- v1.5 Minimum Production Release for controlled direct-customer commercial use.
- v2.0 Multi-Tenant SaaS Commercial Release for customer-of-customer SaaS commercialization.

v1.5 and v2.0 have different commercial boundaries. v1.5 targets controlled production use for direct customers through managed, single-tenant, limited production, or pilot commercial deployments. v2.0 targets a multi-tenant SaaS model in which customers can offer the service to their own customers, with tenant isolation, organization management, permissions, billing, audit, SLA, compliance, and operations readiness.

These are future roadmap and readiness targets. They do not describe capabilities already present in v0.9.

## Roadmap Boundary

The roadmap extends the project plan without implementing features. It clarifies the expected path:

- v0.9 remains a Douyin Provider POC / Sandbox Integration stage.
- v1.0 targets small-scope user testing of real Douyin authorization, publishing, status tracking, and minimum metrics read under explicit user authorization.
- v1.1 through v1.4 harden real integration, publishing, metrics, and production release candidate readiness.
- v1.5 targets Minimum Production Release readiness for direct customers.
- v1.6 through v1.9 build the multi-tenant SaaS foundation, access control, billing, operations, and release candidate readiness.
- v2.0 targets Multi-Tenant SaaS Commercial Release readiness for customer-of-customer commercialization.

## Commercial Boundary

v1.5 is a controlled direct-customer commercial target. It requires production deployment documentation, customer acceptance, security and privacy boundaries, backup and restore, monitoring, incident response, support workflow, OAuth / token / provider reliability, publish workflow reliability, metrics reliability, operational runbooks, and legal / commercial boundary documentation.

v1.5 is not a multi-tenant SaaS release. It does not default to customer-of-customer onboarding, white-label resale, marketplace distribution, unlimited scale, or SLA commitments unless future documentation and contracts explicitly establish them.

v2.0 is the target for multi-tenant SaaS commercialization. It requires strong tenant isolation, tenant / customer / customer-of-customer data boundaries, organization and RBAC models, tenant-scoped provider credentials, audit logs, billing / plan / quota enforcement, admin console, support operations, SLA readiness, privacy / compliance documentation, tenant-level monitoring, security incident response, scalability, and ongoing platform policy compliance.

## Non-Goals

This batch does not:

- Add backend API routes.
- Modify database tables or migrations.
- Add frontend UI.
- Connect to real Douyin.
- Implement OAuth.
- Create OAuth URLs.
- Add OAuth callback routes.
- Add OAuth state storage.
- Add token exchange.
- Store token, secret, API key, credential, authorization code, or OAuth state values.
- Read real environment secrets.
- Call external services.
- Fetch real metrics.
- Upload, publish, or schedule publishing.
- Add real tenant, billing, RBAC, or admin console implementation.
- Declare the v0.9 POC complete.
- Declare v1.0, v1.5, or v2.0 complete.

## Consequences

The project now has an explicit documentation bridge from the v0.9 sandbox POC foundation to future user test, production, and SaaS milestones. Future implementation batches can reference clear release boundaries and readiness checklists instead of mixing commercial claims into POC work.

The documentation also makes the commercialization gap visible: v1.5 may support controlled direct-customer commercial use only after its readiness criteria are satisfied, while v2.0 is the target that can support customer-of-customer SaaS commercialization after multi-tenant SaaS readiness is satisfied.

## Follow-up Work

- Define and execute v1.0 Douyin Integration User Test Release implementation ADRs.
- Define real OAuth callback, OAuth state, token lifecycle, encrypted credential storage, and provider adapter ADRs before real platform integration.
- Define v1.5 production deployment, support, monitoring, incident response, privacy, and customer acceptance checklists before controlled direct-customer commercial use.
- Define v2.0 tenant, RBAC, billing, audit, compliance, operations, and SLA ADRs before customer-of-customer SaaS commercialization.
