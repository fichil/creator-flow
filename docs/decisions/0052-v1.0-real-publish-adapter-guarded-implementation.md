# ADR 0052: v1.0 Real Publish Adapter Guarded Implementation

## Status

Accepted for v1.0 Batch 7 guarded foundation.

## Context

v1.0 is the Douyin Integration User Test Release target, but real Douyin runtime behavior remains unavailable. Batch 5 added real provider feature flag / kill switch controls, and Batch 6 added the user-confirmed publish intent workflow.

Batch 7 establishes a guarded real publish adapter foundation before any future real provider publish execution can be considered. It is a local guarded attempt layer, not a real Douyin publish implementation.

## Decision

Batch 7 adds an internal guarded publish adapter boundary and a metadata-only `publish_attempts` schema. A guarded publish attempt must depend on a Batch 6 confirmed publish intent, must pass review / media / metadata / credential reference preflight, and must pass Batch 5 real provider controls.

`douyin_real` remains blocked by default. Tests may use obvious fake control policy objects to exercise guard categories, but the implementation still does not call external networks, upload media, publish to Douyin, schedule publishing, fetch metrics, or persist provider response material.

Publish attempt output must be metadata-only: local attempt id, publish intent id, review item id, provider id, source type, guard status, external call status, timestamps, safe status message, and status-change reason. Provider response, upload response, publish response, raw request, raw response, token, secret, credential, authorization code, and raw OAuth state material must not be persisted, returned, logged, audited, or placed in fixtures.

`fake_local`, `douyin_sandbox`, and `douyin_real` remain strictly separated. `douyin_real` must not fallback to `douyin_sandbox`.

## Explicit Non-Goals

Batch 7 does not:

- Execute real publishing.
- Upload video.
- Schedule publishing.
- Call Douyin API or any business external service.
- Fetch real metrics.
- Add OAuth start routes.
- Add OAuth callback routes.
- Create OAuth URLs.
- Implement real OAuth.
- Implement real token exchange.
- Save token, secret, credential, authorization code, raw OAuth state, cookie, session, API key, bearer, raw request, or raw response material.
- Store provider response, upload response, publish response, external response, or Douyin response payloads.
- Add real token storage or real credential storage.
- Add frontend OAuth UI.
- Enable `douyin_real`.
- Modify the `v0.9.0` tag.
- Declare v1.0, v1.5, or v2.0 complete.
- Declare production, commercial, or SaaS readiness.

## Safe Guard Rules

The guarded adapter must reject missing publish intents, non-confirmed / cancelled / blocked intents, non-ready review items, missing local media artifacts, invalid metadata, missing credential references, duplicate active attempts, unsupported providers, real provider disabled states, active kill switch states, missing platform preconditions, and sandbox fallback attempts.

The external call status must remain `not_called` or `blocked` in this batch. No provider publish call path is enabled by default.

## Dependency Gates

- Batch 6 confirmed publish intent is required before any guarded attempt.
- Batch 5 real provider controls must continue to block `douyin_real`.
- Batch 4 credential reference boundary must exist before real-provider credential readiness can be evaluated.
- Batch 8 is required before publish status reconciliation.
- Batch 9 is required before limited real metrics read.
- Future real publish execution requires a separate accepted ADR, tests, security scan, platform preconditions, and explicit enablement.

## Consequences

Future real publishing must pass through a human-confirmed publish intent and this guarded adapter boundary or an equivalent reviewed gate. Batch 7 records local guarded attempt metadata only and keeps real provider publishing unavailable.
