# ADR 0006: 归档先于删除

## 状态

Accepted

## 背景

v0.1 Local Runnable Skeleton 已支持本地创建内容项目和显式导入素材。第二批实现需要补充基础项目管理能力，但仍处于本地骨架阶段，尚未提供完整审核、生成、渲染或发布工作流。

项目一旦包含用户显式导入的素材，物理删除会带来误删风险，也会让后续调试、回归测试和用户追溯更困难。上传文件同样属于用户运行时数据，当前没有完整回收站、引用检查或删除确认流程。

## 决策

当前只实现 `archive`，不实现物理删除项目。

归档项目会将 `ContentProject.status` 设置为 `archived`，但不会删除 SQLite 记录，不会删除 `UserMaterial` 记录，也不会删除 `uploads/` 下的文件。项目列表默认隐藏 archived 项目，用户显式选择“显示归档项目”后仍可查看。

已归档项目禁止继续导入素材。归档表示项目已经从当前工作流中退出，继续添加素材会让状态语义变得模糊，也会增加后续恢复、生成或审核流程的复杂度。

## 结果

- 用户可以把不再活跃的项目从默认列表中移走。
- 已导入素材和文件引用仍然可追溯。
- 本阶段避免实现不可逆的项目删除和文件删除。
- 后续如需删除能力，应先设计清晰的确认流程、文件引用检查和运行时数据清理策略。
- archived 项目仍然可以作为只读历史项目查看，不能继续添加素材。
