# ADR 0039: v0.9 Douyin Provider Sandbox Metrics / Mock Workflow POC

Status: Accepted

## Context

v0.9 Batch 1 introduced backend-only Douyin adapter skeletons for `douyin_sandbox` and `douyin_real`. v0.9 Batch 2 added deterministic sandbox-only simulated adapter operations for `douyin_sandbox` while keeping `douyin_real` blocked. v0.9 Batch 3 added backend-only registry / factory routing so exact whitelisted provider ids can resolve to descriptors and adapters without fallback.

Batch 4 needs a small backend-only workflow POC that proves the registry / factory route can support a mock account connection, a sandbox metrics read shape, and a dry-run publish shape while preserving all real-platform boundaries.

## Decision

- This batch is v0.9 Batch 4.
- This batch introduces a backend-only Douyin Provider Sandbox Metrics / Mock Workflow POC.
- The workflow uses Batch 3 registry / factory routing to create the `douyin_sandbox` adapter.
- The workflow returns deterministic sandbox-only simulated results for mock account connection, sandbox metrics POC, and dry-run publish.
- `douyin_real` remains blocked / not implemented.
- Unknown providers continue to fail explicitly and must not fallback to sandbox.
- This batch does not connect to real Douyin.
- This batch does not implement OAuth.
- This batch does not create OAuth URLs.
- This batch does not add backend APIs.
- This batch does not modify the database.
- This batch does not add frontend UI.
- This batch does not store tokens, secrets, credentials, authorization codes, or OAuth state.
- This batch does not call external services.
- This batch does not upload, publish, schedule, or fetch real metrics.
- This batch does not declare v0.9 POC complete.

## Sandbox Workflow Boundary

The sandbox workflow is local contract simulation only. It may return stable fake ids such as `sandbox_connection_001`, `sandbox_account_001`, `sandbox_metrics_snapshot_001`, `sandbox_video_001`, and `sandbox_publish_001`.

The workflow must not use real current time, randomness, real user data, real platform data, callback payloads, authorization material, or external service responses. Fixed timestamps are allowed only as deterministic test values.

## Mock Account Connection Boundary

Mock account connection may report a deterministic `simulated_connected` sandbox result. It does not represent a real connected Douyin account, real OAuth login, real account authorization, token exchange, or account verification.

## Sandbox Metrics Boundary

Sandbox metrics POC may return deterministic mock metrics for contract testing. These values must be labeled as sandbox / simulated / dry-run and must not be represented as real Douyin metrics or user account performance.

## Dry-Run Publish Boundary

Dry-run publish may return a deterministic simulated publish result with sandbox fake ids. It does not upload video, publish content, schedule content, change platform state, or prove that real publishing is available.

## Real Provider Boundary

`douyin_real` remains a future real provider placeholder. It must continue returning blocked / not implemented operation results and must not return success, read configuration, create authorization URLs, perform token exchange, fetch metrics, upload, publish, schedule, or call external services.

## Security Requirements

- Do not call real Douyin APIs.
- Do not call external HTTP or SDK services.
- Do not read environment secrets.
- Do not read credentials.
- Do not read or write tokens.
- Do not save authorization codes or OAuth state.
- Do not create OAuth URLs.
- Do not add backend API routes.
- Do not modify database tables or migrations.
- Do not add frontend UI.
- Keep sandbox outputs deterministic and clearly fake.
- Keep tests explicit about no network calls, no sensitive value leakage, source separation, and unknown-provider failure.

## Non-Goals

- 不接真实 Douyin API。
- 不实现 OAuth。
- 不创建 OAuth URL。
- 不新增 OAuth callback route。
- 不新增 OAuth state storage。
- 不新增 token exchange。
- 不新增 token storage。
- 不新增 credential storage。
- 不保存 token。
- 不保存 secret。
- 不保存 API key。
- 不保存 credential。
- 不保存 authorization code。
- 不保存 OAuth state。
- 不读取真实环境变量密钥。
- 不抓取真实指标。
- 不上传。
- 不发布。
- 不排期发布。
- 不调用外部服务。
- 不新增 backend API。
- 不修改数据库表。
- 不新增前端 UI。
- 不声明 v0.9 POC 已完成。

## Consequences

Batch 4 gives future real Provider, OAuth, metrics, and publish workflow design a stable adapter contract and workflow-level test boundary. It proves that provider selection, sandbox adapter behavior, mock connection shape, mock metrics shape, and dry-run publish shape can be tested without external services or sensitive material.

This decision does not make v0.9 POC complete and does not make any real Douyin capability available.
