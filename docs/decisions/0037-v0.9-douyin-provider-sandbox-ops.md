# ADR 0037: v0.9 Douyin Provider Sandbox Operation Simulation

Status: Accepted

## Context

v0.9 Batch 1 introduced a backend-only Douyin Provider Adapter Skeleton for `douyin_sandbox` and `douyin_real`. All operations initially returned blocked / not implemented boundary results so the codebase could establish adapter shape without enabling real platform behavior.

v0.9 Batch 2 needs a deterministic, dependency-free sandbox simulation layer to exercise the adapter contract before any real Provider, OAuth, token lifecycle, metrics, upload, publish, or scheduling work is considered.

## Decision

- This batch is v0.9 Batch 2.
- This batch only implements sandbox-only operation simulation for `douyin_sandbox`.
- `douyin_real` remains blocked / not implemented for every operation.
- Sandbox results must be deterministic and use stable fake ids.
- Sandbox results must be dry-run only and must not create real side effects.
- This batch does not connect to real Douyin.
- This batch does not implement OAuth.
- This batch does not add backend APIs.
- This batch does not modify the database.
- This batch does not add frontend UI.
- This batch does not store tokens, secrets, credentials, authorization codes, or OAuth state.
- This batch does not call external services.
- This batch does not upload, publish, schedule, or fetch real metrics.
- This batch does not declare v0.9 POC complete.

## Sandbox Operation Boundary

`douyin_sandbox` may return simulated success results for adapter operations. These results are safe local contract outputs only. They must include clear sandbox / simulated / dry-run semantics and stable fake references such as `sandbox_video_001`, `sandbox_publish_001`, or `sandbox_metrics_001`.

Sandbox simulation must not use real current time, randomness, user data, platform data, raw callback payloads, real authorization material, or external service responses. If future simulations need time-like fields, those fields must be fixed or explicitly supplied as test input.

## Real Provider Boundary

`douyin_real` must continue returning blocked / not implemented boundary results. It must not return success, generate authorization URLs, read configuration, read credentials, perform token exchange, fetch metrics, upload, publish, schedule, or call external services.

## Security Boundary

The sandbox simulation layer must not read environment secrets, access local credential files, store sensitive material, or persist operation results. Operation results must avoid fields designed to carry sensitive values and must continue to report that no external call, credential read, token read, or token write occurred.

## Non-Goals

- 不接真实 Douyin API。
- 不实现 OAuth。
- 不新增 backend API。
- 不新增 OAuth callback route。
- 不新增 OAuth state storage。
- 不新增 token exchange。
- 不新增 token storage。
- 不新增 credential storage。
- 不修改数据库表。
- 不新增前端 UI。
- 不保存 token / secret / API key / credential / authorization code / OAuth state。
- 不读取真实环境变量密钥。
- 不调用外部服务。
- 不抓取真实指标。
- 不上传。
- 不发布。
- 不排期发布。
- 不声明 v0.9 POC 已完成。

## Consequences

Batch 2 gives future provider work a stable adapter contract and test boundary while keeping all real platform capabilities blocked. Backend tests can verify sandbox operations, source separation, deterministic fake ids, absence of network calls, and absence of sensitive environment value reads.

This creates a safer stepping stone for later ADRs covering sandbox/mock callback smoke tests, provider status dry-run wiring, encrypted credential storage, token exchange / refresh / revoke, real metrics read POC, and any upload / publish / scheduling workflow.
