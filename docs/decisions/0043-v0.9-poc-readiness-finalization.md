# ADR 0043: v0.9 Douyin Provider POC Readiness Finalization

Status: Accepted

## Context

v0.9 Batch 1 through Batch 7 established the Douyin provider sandbox POC surface: adapter skeleton, sandbox-only operation simulation, provider registry / factory routing, sandbox metrics / mock workflow, sandbox-only backend API contract, and frontend sandbox POC panel.

The project now needs a release candidate review package that gathers validation commands, checklist coverage, test matrix, smoke boundaries, safety scans, and documentation consistency before any human review, PR, merge, tag, or release decision.

## Decision

v0.9 Batch 8 is POC readiness finalization / release candidate preparation.

It adds the v0.9 RC checklist, v0.9 test matrix, validation script, ADR 0043, and documentation consistency updates. It does not add real platform capability and does not declare v0.9 POC complete.

## Validation Boundary

The validation package covers:

- backend tests,
- frontend tests,
- frontend build,
- smoke API checks,
- `git diff --check`,
- `git status --short`,
- documentation wording scans,
- sensitive field scans,
- sandbox keyword scans,
- manual review notes for warnings.

The validation script runs only local checks. It does not modify files, commit, push, create tags, merge branches, create releases, read real secrets, or contact real Douyin.

## Scope

- Add `docs/releases/v0.9-douyin-provider-poc-rc-checklist.md`.
- Add `docs/testing/v0.9-douyin-provider-poc-test-matrix.md`.
- Add `scripts/validate-v0.9-poc.ps1`.
- Update README, roadmap, architecture, product spec, development docs, roadmap-to-v2 docs, and the v0.9 readiness checklist.
- Keep the package ready for human review / PR / merge / release decision.

## Non-Goals

- No real Douyin feature.
- No real OAuth.
- No OAuth URL generation.
- No OAuth callback route.
- No OAuth state storage.
- No token exchange.
- No token storage.
- No credential storage.
- No database schema change.
- No real Provider implementation.
- No substantive business workflow.
- No real metrics fetching.
- No upload, publish, or scheduling.
- No tenant, billing, RBAC, or admin console implementation.
- No final v0.9 release declaration.
- No v1.0, v1.5, or v2.0 completion declaration.

## Consequences

The v0.9 POC has a repeatable RC review package, but release remains a human decision. Batch 8 makes readiness auditable without increasing the platform integration surface.

The next step is human review / PR / merge / tag decision. It is not automatic release, automatic tag creation, automatic merge to main, or GitHub Release creation.

## Future Work

- Final v0.9 release checklist and tag decision, if human review accepts the RC package.
- v1.0 Douyin Integration User Test Release planning.
- Separate ADRs for real OAuth, callback, OAuth state, encrypted credential storage, token exchange / refresh / revoke, real metrics read, publish workflow, disconnect / revoke lifecycle, and feature disablement.
- v1.5 remains the future Minimum Production Release target.
- v2.0 remains the future Multi-Tenant SaaS Commercial Release target.
