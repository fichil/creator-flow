# ADR 0055: v1.0 User Test Readiness Release Candidate

## Status

Accepted for v1.0 Batch 10 user test readiness / release candidate package.

## Context

v1.0 is the Douyin Integration User Test Release line. Batches 0-9 established
local-only, metadata-only, and guarded foundations for planning, OAuth boundary,
OAuth state, token exchange boundary, credential reference boundary, real provider
controls, user-confirmed publish intent, guarded publish attempts, publish status
reconciliation, and limited metrics snapshots.

Batch 10 is a release candidate readiness package for small-scope internal /
trusted user testing.

It is not a formal v1.0 release tag and does not make the app production,
commercial, or SaaS ready.

## Review Notes

This ADR records a release-candidate readiness decision only.

It does not authorize final release, production rollout, commercial readiness,
SaaS readiness, tag creation, or real Douyin runtime work.

The accompanying RC checklist, release notes draft, PR draft, user test guide,
rollback runbook, and test matrix must continue to preserve that boundary.

## Decision

Batch 10 creates the v1.0 user test readiness / release candidate package. The
package must collect the Batch 0-9 boundary decisions, validation expectations,
risk statements, release notes draft, PR description draft, user test guide,
rollback / disablement runbook, RC checklist, and RC test matrix.

Batch 10 does not add real Douyin provider runtime. It does not implement real
OAuth, create OAuth URLs, add OAuth start routes, add OAuth callback routes,
implement real token exchange, store real token / secret / credential /
authorization code / raw OAuth state material, call Douyin API, call business
external services, upload video, publish, schedule publishing, query real Douyin
status, or read real Douyin metrics.

The Batch 10 package must confirm that all real external capabilities remain
blocked by default, `douyin_real` remains disabled, `douyin_real` cannot fallback
to `douyin_sandbox`, and all local-only / metadata-only schema remain free of raw
token, secret, credential, authorization code, raw OAuth state, provider
response, publish response, status response, metrics response, external response,
and Douyin response material.

The package must also confirm that UI and docs do not imply real Douyin
authorization, real upload, real publish, real status query, or real metrics read
is available.

## Required Artifacts

- RC checklist.
- Draft release notes.
- Draft PR description.
- User test guide.
- Rollback / disablement runbook.
- RC test matrix.
- Validation script or documented validation commands.
- Documentation updates referencing ADR 0055 and the RC package.

## Explicit Non Goals

Batch 10 does not:

- Create a formal v1.0 tag.
- Create or modify a `v1.0.0` tag.
- Enable real Douyin provider runtime.
- Implement real OAuth.
- Create OAuth URLs.
- Add OAuth start routes.
- Add OAuth callback routes.
- Implement real token exchange.
- Store token, secret, credential, authorization code, raw OAuth state, cookie,
  session, API key, bearer, raw request, raw response, provider response, upload
  response, publish response, status response, metrics response, external
  response, or Douyin response material.
- Call Douyin API or business external services.
- Upload video.
- Execute real publishing.
- Schedule publishing.
- Query real Douyin status.
- Read real Douyin metrics.
- Declare v1.0, v1.5, or v2.0 finished.
- Declare production, commercial, or SaaS readiness.

## Consequences

The repository can present a coherent v1.0 user test release candidate package
for human review while preserving the same safety boundaries as Batch 0-9.
Future real integration still requires separate accepted implementation work for
real OAuth, credential storage, platform app review, permissions, rate limits,
audit, and security validation.
