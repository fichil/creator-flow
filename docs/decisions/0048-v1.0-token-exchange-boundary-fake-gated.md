# ADR 0048: v1.0 Token Exchange Boundary With Fake-Gated Integration

Status: Accepted

## Context

ADR 0046 defined the future OAuth callback contract. ADR 0047 added the internal OAuth state storage and anti-replay foundation required before any token exchange boundary can proceed.

v1.0 Batch 3 establishes a token exchange boundary with fake-gated integration. It prepares the safety contract for future real token exchange without making real Douyin token exchange available.

## Decision

v1.0 Batch 3 adds an internal-only token exchange boundary service and backend tests. The boundary depends on Batch 2 OAuth state storage / anti-replay foundation. Without valid single-use state consumption, fake-gated exchange simulation must not proceed.

The token exchange result must be metadata-only and redacted. It may report safe status, provider id, source type, state validation status, whether token material would have been received, whether credential storage remains required, whether an external exchange was performed, and whether sandbox fallback occurred. It must not return token material.

The fake-gated simulator may use obvious fake authorization code values in tests, but the service must not expose fake token values and must not return, persist, log, audit, or display authorization code, raw OAuth state, access token, refresh token, secret, credential, cookie, session, raw request, or raw response values.

`fake_local`, `douyin_sandbox`, and `douyin_real` must remain strictly separated. `douyin_real` is blocked / disabled by default and must not fallback to `douyin_sandbox`.

## Security Boundary

Batch 3 does not:

- Add an OAuth start route.
- Add an OAuth callback route.
- Create an OAuth URL.
- Implement real OAuth.
- Implement real token exchange.
- Call Douyin APIs or business external services.
- Read real client secrets.
- Store authorization codes.
- Store access tokens.
- Store refresh tokens.
- Store token values.
- Store secrets.
- Store credential material.
- Store cookies or sessions.
- Store raw OAuth state.
- Add credential storage.
- Add frontend OAuth UI.
- Enable `douyin_real`.

## Dependency Gates

- Batch 2 state validation must run before fake-gated exchange simulation.
- Batch 3 only provides fake-gated token exchange boundary behavior.
- Credential reference and encrypted storage must wait for Batch 4.
- Real provider runtime enablement must wait for Batch 5 feature flag / kill switch controls.

## Consequences

The project gains a reviewable token exchange boundary and tests for state dependency, replay rejection, expiry rejection, provider mismatch rejection, unsupported provider failure, real provider disablement, sandbox fallback prevention, redaction, and no external service calls.

Batch 3 does not make v1.0 complete, does not make v1.5 or v2.0 complete, and does not claim production, commercial, or SaaS readiness.
