# ADR 0036: v0.9 Douyin Provider Adapter Skeleton

Status: Accepted

## Context

v0.8.0 has been released as the Provider & Credential Security Foundation. v0.9 Batch 0 then opened Douyin Provider POC / Sandbox Integration with planning, checklist, and ADR-only work. The next safe step is to create a backend-only adapter skeleton that can be reviewed and tested without enabling real platform behavior.

The repository already has Provider Registry, Connection State, Credential Reference, Security Audit, OAuth Boundary, Token Lifecycle Boundary, Readiness Summary, redaction helpers, and `fake_local` / `douyin_sandbox` / `douyin_real` source separation. v0.9 Batch 1 must layer on those boundaries instead of bypassing them.

## Decision

- v0.9 Batch 1 introduces backend-only Douyin Provider Adapter Skeleton.
- It includes `douyin_sandbox` and `douyin_real` skeletons.
- All real operations return blocked / not implemented results.
- No external service calls are allowed.
- No OAuth implementation is introduced.
- No callback route is introduced.
- No token exchange or storage is introduced.
- The skeleton may expose safe local metadata and operation boundary results only.
- Backend tests must verify that operations stay blocked, do not call networks, do not read environment secrets, and do not add API routes.

## Adapter Boundary

The adapter skeleton lives in the backend providers layer. It is not registered as a real client, does not create HTTP clients, does not load platform SDKs, and does not execute platform workflows.

Adapter properties may describe provider id, provider name, source type, real-provider flag, and unsupported capability flags. Operations may exist only to return safe boundary results that make blocked behavior explicit.

## Sandbox Boundary

`douyin_sandbox` is a sandbox skeleton only. It is not real Douyin OAuth, not a real callback handler, not a real token exchange path, and not real metrics read. Any future sandbox/mock callback smoke test requires a separate ADR, branch, tests, and security scan.

## Real Provider Boundary

`douyin_real` remains a future real provider skeleton. The presence of a class or module must not be interpreted as real provider integration, real account connection, real OAuth readiness, real metrics readiness, upload readiness, publish readiness, or scheduling readiness.

## Operation Blocking Boundary

The following operations must return blocked / not implemented boundary results until separate ADRs approve otherwise:

- `start_oauth`
- `handle_oauth_callback`
- `exchange_token`
- `refresh_token`
- `revoke_token`
- `disconnect`
- `fetch_metrics`
- `prepare_publish`
- `upload_video`
- `publish_video`
- `schedule_publish`

Each result must indicate no external call, no credential read, no token read, and no token write. Results must not contain fields designed to carry sensitive values.

## Source Separation

`fake_local`, `douyin_sandbox`, and `douyin_real` must stay distinct:

- `fake_local` remains local fake/demo/test workflow only.
- `douyin_sandbox` remains sandbox/mock or placeholder only.
- `douyin_real` remains future real provider only.

Sandbox data must never be displayed or treated as real data. A sandbox/mock callback must never be represented as real OAuth.

## Security Requirements

- Do not call external services.
- Do not read environment secrets.
- Do not read credentials.
- Do not read or write tokens.
- Do not save authorization codes or OAuth state.
- Do not parse raw platform requests or responses.
- Do not add backend API routes.
- Do not modify database tables.
- Do not add frontend UI.
- Keep tests explicit about blocked behavior and source separation.

## Non-Goals

- 不接真实 Douyin API。
- 不实现 OAuth。
- 不新增 OAuth callback route。
- 不新增 OAuth state storage。
- 不新增 token exchange。
- 不保存 token。
- 不保存 secret。
- 不保存 API key。
- 不保存 credential。
- 不保存 authorization code。
- 不保存 OAuth state。
- 不抓取真实指标。
- 不上传。
- 不发布。
- 不排期发布。
- 不调用外部服务。
- 不新增 backend API。
- 不修改数据库表。
- 不新增前端 UI。

## Consequences

The codebase gains a reviewable backend adapter shape for future v0.9 work without enabling real platform behavior. Tests can now assert that all adapter operations remain blocked and that route, network, credential, token, and source-separation boundaries are preserved.

This ADR does not make v0.9 POC complete. It only establishes the skeleton needed before later sandbox/mock or dry-run work can be proposed.

## Follow-up Work

- 后续可以单独做 sandbox/mock callback smoke test ADR。
- 后续可以单独做 provider adapter dry-run status wiring。
- 后续可以单独做 credential storage encryption ADR。
- 后续可以单独做 real metrics read POC ADR。
- 后续任何真实 Douyin API 调用必须单独批准。
