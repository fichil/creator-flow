# v0.9 Douyin Provider POC Test Matrix

This matrix defines the validation surface for the v0.9 Douyin Provider POC / Sandbox Integration release candidate review package. It is a review aid, not a final release declaration.

## Backend Tests

- `backend/tests/test_douyin_provider_adapter_skeleton.py`: adapter skeleton, blocked operations, no external calls, no sensitive payload.
- `backend/tests/test_douyin_provider_sandbox_ops.py`: deterministic sandbox-only simulated operations for `douyin_sandbox`, with `douyin_real` blocked.
- `backend/tests/test_douyin_provider_registry_routing.py`: exact provider id routing, descriptors, factory behavior, unknown provider no fallback.
- `backend/tests/test_douyin_provider_sandbox_metrics_poc.py`: mock account connection, sandbox metrics preview, publish dry-run workflow.
- `backend/tests/test_douyin_sandbox_api_contract.py`: sandbox API contract, provider list / lookup, sandbox operation endpoints, `douyin_real` boundary, unknown provider boundary.
- Existing backend regression tests: project, materials, planning, fake rendering, fake publishing, fake metrics, provider security foundations, and readiness panels.

## Frontend Tests

- `frontend/src/api/douyinSandbox.test.ts`: sandbox API client paths, mocked fetch, no Douyin external domain, no authorization header.
- `frontend/src/components/DouyinSandboxPanel.test.tsx`: boundary banner, provider descriptor display, mock connection action, metrics preview action, publish dry-run action, error handling, sensitive field guard.
- `frontend/src/pages/ProjectListPage.test.tsx`: project list integration with the sandbox panel and provider metadata panels.
- Existing frontend regression tests: project creation/list/detail, fake planning, fake rendering, fake publishing, fake metrics, provider read-only panels, and readiness UI.

## Smoke Tests

- `scripts/smoke-api.ps1`: health, project create/update, material create, detail read, archive, project list behavior.
- Sandbox provider list: `GET /api/providers/douyin`.
- Sandbox provider lookup: `GET /api/providers/douyin/douyin_sandbox` and `GET /api/providers/douyin/douyin_real`.
- Sandbox mock connection: `POST /api/providers/douyin/sandbox/mock-connection`.
- Sandbox metrics preview: `POST /api/providers/douyin/sandbox/metrics-preview`.
- Sandbox publish dry-run: `POST /api/providers/douyin/sandbox/publish-dry-run`.

## Security Scans

- No real token, refresh token, secret, API key, credential, authorization code, OAuth state, cookie, session, or bearer value.
- No real Douyin response or raw platform payload.
- No SQLite DB, uploads, dist, node_modules, `.venv`, generated media, or runtime files in the commit.
- No external Douyin domain calls in frontend or backend code.
- Sensitive field-name matches must be reviewed as deny-lists, tests, redaction guards, or boundary documentation only.

## Documentation Scans

- No false production or commercial readiness wording.
- v1.0, v1.5, and v2.0 remain future roadmap targets.
- v0.9 remains POC / sandbox only.
- `douyin_sandbox` remains sandbox / simulated / dry-run.
- `douyin_real` remains blocked / not implemented.
- Unknown provider ids do not fallback.

## Manual Review Checklist

- [ ] UI wording does not imply real Douyin connection or publish success.
- [ ] API response shape remains deterministic and sandbox-only.
- [ ] Documentation is consistent across README, roadmap, architecture, product spec, development docs, release checklist, test matrix, and ADRs.
- [ ] Release boundary is clear: RC package prepared, not final release.
- [ ] No real platform data, real credentials, or runtime artifacts are included.
- [ ] Next step is human review / PR / merge / release decision.
