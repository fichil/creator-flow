# v1.0 Publish Status Reconciliation Test Matrix

This matrix covers the v1.0 Batch 8 publish status reconciliation foundation. Batch 9 now adds a separate local-only limited metrics snapshot foundation on top of these status snapshots. Neither matrix requires real Douyin, OAuth URLs, OAuth start routes, OAuth callback routes, real token exchange, real token storage, real credential storage, upload, actual publish, scheduled publish, real status query, real metrics query, or external service calls.

| Scenario | Expected result | Safety checks |
| --- | --- | --- |
| Publish attempt required | Missing attempt returns `publish_attempt_required`. | No reconciliation or snapshot row is created. |
| Metadata-only reconciliation | Valid local attempt creates a reconciliation record. | Record stores safe local status metadata only. |
| Metadata-only status snapshot | Valid local attempt creates a status snapshot. | Snapshot stores safe status, source, timestamps, and safe message only. |
| Duplicate reconciliation prevention | Active reconciliation for the same attempt is rejected. | Existing local reconciliation is not executed again. |
| Stale status ignored | Older observed status returns `stale_status_ignored`. | Newer snapshot remains unchanged. |
| Malformed fixture rejected | Invalid fake or sandbox fixture returns `status_fixture_invalid`. | Fixture raw payload is not persisted or echoed. |
| Real provider disabled | `douyin_real` remains blocked by default. | Batch 5 controls remain blocking. |
| Kill switch active | Kill switch active returns `kill_switch_active`. | Kill switch takes precedence over reconciliation execution. |
| Platform preconditions missing | Missing platform preconditions return `platform_preconditions_missing`. | No real call path opens. |
| Sandbox fallback forbidden | `douyin_real` cannot fallback to `douyin_sandbox`. | Result records no fallback. |
| Unsupported provider | Unknown provider returns `unsupported_provider`. | Unknown id is not mapped to sandbox or fake. |
| No external service | Workflow does not call socket, requests, httpx, urllib, Douyin, or business external services. | Network monkeypatches remain unused. |
| No real status query | External status query request returns `external_status_query_blocked`. | External query status stays `not_called` or `blocked`. |
| No upload | Reconciliation creation does not upload media. | Upload runtime paths remain untouched. |
| No actual publish | Reconciliation creation does not call provider publish. | Existing fake publish execution is not triggered. |
| No scheduled publish | Reconciliation creation does not create scheduled publish records. | No scheduled publish route is added. |
| No OAuth route | Batch 8 adds no OAuth start, callback, URL creation, or token exchange route. | Route list has no new OAuth runtime endpoints. |
| No OAuth URL | UI and API do not open or create authorization URLs. | No browser navigation or URL payload is produced. |
| No token exchange | No exchange runtime is invoked. | No authorization code or token material is returned or persisted. |
| No credential storage | No real credential storage is enabled. | Reconciliation schema has no credential columns. |
| No metrics read | Reconciliation creation does not fetch metrics. | Metrics routes are not invoked by status reconciliation. |
| Batch 9 dependency | Batch 9 metrics snapshots require Batch 8 status snapshots. | Batch 9 does not convert Batch 8 status metadata into real Douyin metrics or real status queries. |
| UI wording | UI says local status reconciliation / publish status snapshot. | UI does not claim real Douyin status was fetched or real publish success was confirmed by Douyin. |
| Redaction | Responses, messages, fixtures, and repr output omit sensitive material values. | No token, secret, credential, authorization code, raw OAuth state, cookie, session, API key, bearer, raw request, raw response, provider response, upload response, publish response, status response, external response, Douyin response, or metrics response values. |
