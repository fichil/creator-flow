# v1.0 Credential Storage Boundary Test Matrix

This matrix covers the v1.0 Batch 4 internal credential reference and encrypted storage boundary. It does not require real Douyin, OAuth URLs, OAuth start routes, OAuth callback routes, real token exchange, real token storage, real credential storage, frontend OAuth UI, publish, upload, metrics, or external service calls.

| Scenario | Expected result | Safety checks |
| --- | --- | --- |
| Metadata-only credential reference | Safe Batch 3 fake-gated exchange metadata creates `credential_reference_created` or a safe disabled result. | Reference is metadata-only; no token material is returned or persisted. |
| Raw token rejected | Raw access token or refresh token input returns `credential_material_rejected`. | Material is not echoed in response, repr, logs, audit, or DB metadata. |
| Raw authorization code rejected | Raw authorization code input returns `credential_material_rejected`. | Code is not persisted or displayed. |
| Raw OAuth state rejected | Raw state input returns `credential_material_rejected`. | Raw state is not logged, audited, displayed, or stored. |
| Secret / credential material rejected | Secret, client secret, credential, cookie, session, bearer, raw request, or raw response input returns `credential_material_rejected`. | No material value appears in safe messages, fixtures, or DB metadata. |
| Unsupported provider | Unknown provider returns `unsupported_provider`. | Provider input is not used to select a default provider. |
| Real provider disabled | `douyin_real` returns `real_provider_disabled`. | No reference is created and no storage is enabled. |
| Sandbox fallback forbidden | `douyin_real` never falls back to `douyin_sandbox`. | Result records no sandbox fallback. |
| Encryption provider unavailable | Encrypted storage required without accepted encryption provider returns `encryption_provider_unavailable`. | No pseudo-encryption, encrypted payload table, or credential material persistence is introduced. |
| No external service | Service does not call network functions. | Socket, URL, requests, and httpx calls stay unused in tests. |
| No token material returned | Result omits authorization code, raw state, access token, refresh token, token value, secret, credential, cookie, session, bearer, raw request, and raw response values. | Dataclass, repr, safe messages, fixtures, and snapshots remain redacted. |
| No real token storage | No token storage table or token material columns are added. | Existing metadata-only tables remain metadata-only. |
| No real credential storage enabled | Metadata-only reference may be created, but real credential storage remains disabled. | No encrypted payload table is added; no fake payload is exposed. |
| No OAuth route | Batch 4 adds no OAuth start, callback, URL, or token exchange route. | App routes do not include OAuth start, callback, URL creation, or token exchange paths. |
| No frontend OAuth UI | Batch 4 adds no frontend OAuth UI. | Frontend remains unchanged for real OAuth entry. |
