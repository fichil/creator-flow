# v1.0 OAuth State Storage Test Matrix

This matrix covers the v1.0 Batch 2 internal OAuth state storage and anti-replay foundation. As of Batch 4, fake-gated token exchange boundary tests may consume states through this foundation and credential storage boundary tests may rely on safe Batch 3 metadata, but this matrix still does not require real Douyin, OAuth URLs, OAuth callback routes, real token exchange, real token storage, real credential storage, frontend OAuth UI, publish, upload, metrics, or external service calls.

| Scenario | Expected result | Safety checks |
| --- | --- | --- |
| Create state | A known provider creates pending state metadata. | Raw state is returned only internally and is not persisted. |
| Digest-only persistence | Database stores `state_digest` and safe metadata only. | No raw state, authorization code, token, secret, credential, cookie, session, raw request, or raw response column exists. |
| One-time consume | A valid pending state for the matching provider becomes consumed. | `consumed_at` is set and no token exchange occurs. |
| Replay reject | A second consume of the same state is rejected. | Replay result is safe and does not reveal raw state. |
| Expired reject | Expired state is rejected and marked expired. | Expiry status is safe and redacted. |
| Missing state reject | Missing state returns an invalid missing-state category. | No digest lookup leaks sensitive values. |
| Malformed state reject | Malformed or unknown state returns an invalid malformed-state category. | No raw callback payload is stored. |
| Provider mismatch reject | State created for one provider cannot be consumed by another provider. | `douyin_real` does not fallback to `douyin_sandbox`. |
| Unsupported provider reject | Unknown provider returns `unsupported_provider`. | Error message is safe and does not echo sensitive-looking provider input. |
| Source separation | `fake_local`, `douyin_sandbox`, and `douyin_real` keep registry source types. | Source values remain `fake_local`, `sandbox`, and `real`. |
| Redaction expectations | Safe messages omit raw state and sensitive values. | Tests assert no token, secret, credential, authorization code, cookie, session, API key, bearer, raw request, or raw response value leaks. |
| No external service | State creation and consume do not call network functions. | Socket, URL, requests, and httpx calls stay unused in tests. |
| No OAuth route | Batch 2 adds no start or callback route. | App routes do not include OAuth start, callback, or token exchange paths. |
| No real token exchange | State validation can be reused by the Batch 3 fake-gated boundary, but Batch 2 itself stops at state validation. | No real token exchange route, token storage, credential storage, or stored token is introduced. |
| No real credential storage | State foundation does not store credential material. | Batch 4 may create metadata-only credential references, but no real credential storage or encrypted payload storage is enabled. |
