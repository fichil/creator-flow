# v1.0 Token Exchange Boundary Test Matrix

This matrix covers the v1.0 Batch 3 internal token exchange boundary and fake-gated simulator. As of Batch 4, safe Batch 3 metadata can feed the separate internal credential storage boundary. This matrix still does not require real Douyin, OAuth URLs, OAuth start routes, OAuth callback routes, real token exchange, real token storage, real credential storage, frontend OAuth UI, publish, upload, metrics, or external service calls.

| Scenario | Expected result | Safety checks |
| --- | --- | --- |
| Valid fake-gated exchange | Known fake or sandbox-safe provider with valid consumed state and obvious fake authorization code returns `exchange_simulated`. | Result is metadata-only; no token material is returned or persisted. |
| Missing authorization code | Missing code returns `authorization_code_missing`. | State remains unconsumed and no token exchange is attempted. |
| Malformed authorization code | Non-fake or malformed code returns `authorization_code_malformed`. | Code is not echoed in responses, logs, audit, or fixtures. |
| Missing state | Missing state returns `state_validation_failed`. | No exchange simulation occurs. |
| Malformed state | Malformed state returns `state_validation_failed`. | Raw state is not logged or stored. |
| Replayed state | Reusing consumed state returns `state_validation_failed`. | Replay cannot produce exchange metadata or token material. |
| Expired state | Expired state returns `state_validation_failed`. | Expired state is rejected before exchange simulation. |
| Provider mismatch | State for one provider cannot be used by another provider. | Mismatch does not consume the original provider state and does not fallback. |
| Unsupported provider | Unknown provider returns `unsupported_provider`. | Provider input is not used to select a default provider. |
| Real provider disabled | `douyin_real` returns `real_provider_disabled`. | State remains unconsumed and no external exchange occurs. |
| Sandbox fallback forbidden | `douyin_real` never falls back to `douyin_sandbox`. | Result records no sandbox fallback. |
| No external service | Service does not call network functions. | Socket, URL, requests, and httpx calls stay unused in tests. |
| No token material returned | Result omits authorization code, raw state, access token, refresh token, token value, secret, credential, cookie, session, raw request, and raw response values. | Dataclass, repr, safe messages, fixtures, and snapshots remain redacted. |
| No token storage | No token storage table or token material columns are added. | Existing metadata-only tables remain metadata-only. |
| No real credential storage | Boundary reports credential storage as future required work. | Batch 4 may create metadata-only references, but no real credential storage or encrypted payload storage is enabled. |
| No OAuth route | Batch 3 adds no OAuth start, callback, URL, or token exchange route. | App routes do not include OAuth start, callback, URL creation, or token exchange paths. |
| No frontend OAuth UI | Batch 3 adds no frontend OAuth UI. | Frontend remains unchanged for real OAuth entry. |
