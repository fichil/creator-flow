# v1.0 Real Provider Feature Flag / Kill Switch Test Matrix

This matrix covers the v1.0 Batch 5 internal real provider feature flag / kill switch foundation. It does not require real Douyin, OAuth URLs, OAuth start routes, OAuth callback routes, real token exchange, real token storage, real credential storage, frontend OAuth UI, publish, upload, metrics, or external service calls.

| Scenario | Expected result | Safety checks |
| --- | --- | --- |
| Real provider default blocked | `douyin_real` returns a blocked real-provider decision by default. | No real provider runtime is enabled. |
| Feature flag disabled | Disabled flag returns `blocked_feature_flag_disabled` or equivalent. | No external service is allowed. |
| Kill switch active | Active kill switch returns `blocked_kill_switch_active` or equivalent. | Kill switch blocks even when feature flag is enabled. |
| Kill switch precedence over feature flag | Enabled flag plus active kill switch stays blocked. | Decision message is safe and contains no sensitive material. |
| Missing flag | Missing feature flag returns safe blocked result. | Missing config does not fallback to enabled. |
| Malformed flag | Malformed feature flag returns safe blocked result. | Malformed config does not fallback to enabled. |
| Unsupported provider | Unknown provider returns `blocked_unsupported_provider` or equivalent. | Unknown provider is not mapped to `douyin_sandbox`. |
| Unsupported capability | Unknown capability returns `blocked_operation_not_allowed` or equivalent. | No real provider runtime is enabled. |
| Platform preconditions missing | Missing preconditions return `blocked_preconditions_missing` or equivalent. | Platform readiness must be explicit before later batches can proceed. |
| Sandbox fallback forbidden | `douyin_real` with sandbox fallback attempt returns `blocked_sandbox_fallback_forbidden`. | Result records no sandbox fallback. |
| Fake / sandbox source separation | `fake_local` returns fake-only safe decision; `douyin_sandbox` returns sandbox-only safe decision. | Real feature flag state does not affect fake or sandbox behavior. |
| No external service | Service does not call network functions. | Socket, URL, requests, and httpx calls stay unused in tests. |
| No sensitive material returned | Result omits token, access token, refresh token, secret, client secret, credential, authorization code, raw OAuth state, OAuth state value, cookie, session, API key, bearer, raw request, and raw response values. | Dataclass, repr, safe messages, fixtures, and snapshots remain redacted. |
| No OAuth route | Batch 5 adds no OAuth start, callback, URL, or token exchange route. | App routes do not include OAuth start, callback, URL creation, or token exchange paths. |
| No OAuth URL | Batch 5 creates no OAuth URL. | Tests do not assert any OAuth URL construction. |
| No real token exchange | Real token exchange capability stays blocked. | No external exchange is attempted. |
| No real token storage | Real token storage remains unavailable. | No token storage table or token material persistence is added. |
| No real credential storage | Real credential storage remains unavailable. | No credential material persistence is added. |
| No publish | Real publish capability stays blocked. | Existing fake / sandbox dry-run behavior is not treated as real publish. |
| No metrics | Real metrics capability stays blocked. | Existing fake / sandbox metrics behavior is not treated as real metrics. |
| No frontend OAuth UI | Batch 5 adds no frontend OAuth UI. | Frontend remains unchanged for real OAuth entry. |
