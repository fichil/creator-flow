# v1.0 OAuth Callback Contract Test Matrix

This matrix describes future contract tests for v1.0 OAuth callback implementation. It does not run against real Douyin, does not create OAuth URLs, does not add routes, does not persist OAuth state, does not exchange tokens, and does not store token, secret, credential, authorization code, cookie, session, or OAuth state values.

| Scenario | Future contract expectation | Required safety checks |
| --- | --- | --- |
| Success callback contract | A callback with fake `code` and fake `state` maps to `pending_token_exchange` only after provider support, feature flag, state existence, state freshness, and single-use validation pass. | Response, logs, audit, and snapshots redact fake code/state values and never include real provider values. |
| Cancelled callback contract | User denied / cancelled authorization maps to `authorization_cancelled`. | No token exchange, no credential storage, no retry loop, no sandbox fallback. |
| Provider error callback contract | Provider error maps to `authorization_failed`. | Raw provider error payload is redacted; only safe error class and safe message appear. |
| Missing state | Callback without state maps to `invalid_callback`. | No token exchange, no persistence of callback-sensitive values, no user-facing state value. |
| Malformed state | Malformed or untrusted state maps to `invalid_callback`. | Logs and tests use obvious fake values only. |
| Replayed state | Previously consumed state maps to `invalid_callback`. | State is single-use after Batch 2; replay attempts cannot proceed to token exchange. |
| Expired state | Expired state maps to `invalid_callback`. | Expiry behavior is deterministic and redacted. |
| Unsupported provider | Unknown or unsupported provider id maps to `unsupported_provider`. | Must not fallback to `douyin_sandbox`, `fake_local`, or any default provider. |
| Redaction expectations | All safe responses omit code, state, token, secret, credential, cookie, session, raw request, and raw response values. | Tests assert absence of sensitive field values in responses, logs, audit fixtures, and snapshots. |
| Real provider disabled expectation | Disabled real OAuth maps to `oauth_temporarily_disabled`. | No authorization start continuation, no callback continuation, no token exchange, no credential storage. |
| Sandbox fallback forbidden expectation | `douyin_real` failure or disablement does not fallback to `douyin_sandbox`. | Source separation remains explicit across responses, logs, audit metadata, and fixtures. |

## Fixture Rules

- Use obvious fake values such as `fake_code_for_contract_test`, `fake_state_for_contract_test`, `fake_provider_error`, and `fake_request_id`.
- Do not use values copied from real browser URLs, Douyin callbacks, provider logs, cookies, sessions, token responses, or platform payloads.
- Do not commit generated callback URLs, token payloads, environment values, database files, runtime artifacts, screenshots containing sensitive values, or external service responses.

## Dependency Gate Expectations

- Batch 2 must exist before tests assert persisted state or anti-replay behavior.
- Batch 3 must exist before tests assert token exchange behavior.
- Batch 4 must exist before tests assert credential reference or encrypted storage behavior.
- Batch 5 must exist before tests assert runtime feature flag or kill switch enablement.
