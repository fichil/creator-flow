# v1.0 User Test Guide

<!-- RC Markdown raw verification guard: this file uses real LF line breaks for GitHub raw review. -->


This guide is for internal / trusted test users only.

It supports local v1.0 user test readiness validation and does not validate real
Douyin authorization, real publish, real status query, or real metrics query.

## Guide Review Notes

- This guide is safe for internal and trusted testers only.
- It should be read together with the rollback / disablement runbook.
- It does not ask testers to enter real platform credentials.
- It does not ask testers to validate real Douyin publishing, status, or metrics.

## Target Testers

- Internal maintainers.
- Trusted test users who understand fake / sandbox-safe local data.
- Reviewers checking wording, safety boundaries, and guarded workflow behavior.

## Test Setup Assumptions

- Use a local development environment.
- Use fake / sandbox-safe data only.
- Do not enter real token, secret, credential, authorization code, raw OAuth
  state, cookie, session, API key, or bearer values.
- Do not connect a real Douyin account.
- Do not expect an OAuth URL, upload, publish, status query, or metrics query to run.

## What Testers Can Validate Locally

- Publish intent creation after an approved review item.
- Guarded publish attempt creation from a confirmed publish intent.
- Local status reconciliation snapshot from a guarded publish attempt.
- Local / limited metrics snapshot from a local status snapshot.
- Disabled real provider messaging.
- Freshness visibility for metrics snapshots.
- Permission / platform limitation safe messages.
- Safe handling of duplicate, stale, blocked, malformed, or disabled states.

## What Testers Cannot Validate Yet

- Real Douyin authorization.
- Real OAuth callback behavior.
- Real token exchange or token storage.
- Real credential storage.
- Real upload.
- Real publish.
- Real scheduled publish.
- Real Douyin status query.
- Real Douyin metrics query.

## Safety Instructions

- Do not enter real token, secret, credential, authorization code, OAuth state,
  cookie, session, API key, or bearer values.
- Do not treat sandbox / fake data as real Douyin data.
- Stop testing if any UI, API response, log, fixture, or DB row exposes a
  sensitive value.
- Stop testing if any route or UI appears able to upload, publish, schedule,
  query real status, or query real metrics.
- Stop testing if UI implies real publish, real status, or real metrics happened.
- Stop testing if `douyin_real` appears to fallback to `douyin_sandbox`.

## Expected Screenshots / Evidence

- Local-only publish intent panel.
- Guarded publish attempt panel.
- Local status reconciliation / publish status snapshot panel.
- Local / limited metrics snapshot panel with freshness.
- Disabled real provider safe messaging.
- Permission / platform limitation messaging.
- Any warning or blocked state relevant to the test.
