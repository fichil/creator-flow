# v1.0 Publish Status Reconciliation Contract

This document defines the v1.0 Batch 8 publish status reconciliation boundary. It is not a public commitment that real Douyin status query is available.

Batch 8 creates a local, metadata-only reconciliation and snapshot foundation on top of Batch 7 publish attempts. It does not add real OAuth, OAuth URL creation, OAuth start routes, OAuth callback routes, real token exchange, real token storage, real credential storage, real external Douyin status calls by default, upload, actual publish, scheduled publish, metrics read, or external service calls.

## Boundary

Publish status reconciliation may create a metadata-only local reconciliation record and a metadata-only status snapshot only after an existing publish attempt is found for the same project.

Creating reconciliation does not execute a provider status query. It does not call Douyin, upload media, publish media, schedule publishing, store provider responses, store status responses, or read metrics. The only allowed persisted values are safe local status categories, source type, status source, timestamps, safe messages, and status-change reasons.

## Input Categories

| Input category | Expected boundary handling |
| --- | --- |
| Valid publish attempt | Create metadata-only reconciliation and status snapshot. |
| Missing publish attempt | Return `publish_attempt_required`. |
| Blocked publish attempt | Return `publish_attempt_not_ready` or `status_reconciliation_blocked`. |
| Cancelled publish attempt | Return `publish_attempt_not_ready`. |
| Failed-safe publish attempt | Return `publish_attempt_not_ready`. |
| Duplicate reconciliation request | Return `duplicate_reconciliation`. |
| Stale local status | Return `stale_status_ignored` and keep the newer snapshot. |
| Unsupported provider | Return `unsupported_provider`. |
| Real provider disabled | Return `real_provider_disabled` or a more specific blocking category. |
| Kill switch active | Return `kill_switch_active`. |
| Platform preconditions missing | Return `platform_preconditions_missing`. |
| Sandbox fallback attempt | Return `sandbox_fallback_forbidden`. |
| External status query blocked | Return `external_status_query_blocked`. |
| Malformed fake status fixture | Return `status_fixture_invalid`. |

## Expected Safe Result Categories

- `status_reconciliation_created`
- `status_reconciliation_blocked`
- `publish_attempt_required`
- `publish_attempt_not_ready`
- `duplicate_reconciliation`
- `stale_status_ignored`
- `status_snapshot_created`
- `unsupported_provider`
- `real_provider_disabled`
- `kill_switch_active`
- `platform_preconditions_missing`
- `sandbox_fallback_forbidden`
- `external_status_query_blocked`
- `status_fixture_invalid`

Implementation names may be equivalent, but user-facing and test-visible messages must remain safe and redacted.

## Redaction Rules

User-facing responses must not display token, secret, credential, authorization code, raw OAuth state, cookie, session, API key, bearer, raw request, raw response, provider response, upload response, publish response, status response, external response, Douyin response, or metrics response values.

Logs must not display sensitive material.

Audit records must not display sensitive material.

Database records must not store raw provider response, raw upload response, raw publish response, raw status response, raw external response, raw Douyin response, raw request, raw response, token, secret, credential, authorization code, cookie, session, API key, bearer, raw OAuth state, or metrics response material.

Tests may use only obvious fake values for negative assertions. Test fixtures must not copy real provider callbacks, token responses, cookies, sessions, platform payloads, local secrets, environment secrets, raw request payloads, raw response payloads, or raw Douyin payloads.

## Explicit Non-Goals

Batch 8 does not provide:

- Real OAuth.
- OAuth URL creation.
- OAuth start route.
- OAuth callback route.
- Real token exchange.
- Real token storage.
- Real credential storage.
- Real external Douyin status call by default.
- Upload.
- Actual publish.
- Scheduled publish.
- Metrics read.
- External service call.
- Production, commercial, or SaaS readiness.

## Future Dependency Gates

Any future real status query must be approved separately and must keep provider response redaction, credential readiness, user authorization, platform permissions, auditability, rate-limit handling, and kill switch behavior outside the Batch 8 local-only foundation. Batch 9 must define limited real metrics read before any real metrics fetch exists.
