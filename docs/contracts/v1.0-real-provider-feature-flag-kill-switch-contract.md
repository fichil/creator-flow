# v1.0 Real Provider Feature Flag / Kill Switch Contract

This document defines the internal-only real provider feature flag / kill switch boundary for v1.0 Batch 5. It is not a public API contract.

Batch 5 does not add an OAuth start endpoint, OAuth callback endpoint, OAuth URL creation, real token exchange, real token storage, real credential storage, frontend OAuth UI, external service call, publish workflow, upload workflow, or metrics read. Batch 5 also does not enable real provider runtime behavior.

## Internal-Only Boundary

The boundary exists so future real-provider code must pass a safe control decision before attempting any real provider behavior. `douyin_real` remains disabled / blocked by default. Unknown, missing, malformed, disabled, revoked, expired, or policy-denied control states must block real provider behavior.

`fake_local`, `douyin_sandbox`, and `douyin_real` must remain strictly separated. `douyin_real` must not fallback to `douyin_sandbox`.

## Input Categories

| Input category | Expected boundary handling |
| --- | --- |
| Known real provider with feature flag disabled | Return `feature_flag_disabled` or equivalent blocked category. |
| Known real provider with kill switch active | Return `kill_switch_active`; kill switch takes precedence over an enabled flag. |
| Known real provider with missing flag | Return safe blocked result. |
| Known real provider with malformed flag | Return safe blocked result. |
| Known real provider with unsupported capability | Return `operation_not_allowed` or equivalent blocked category. |
| Known real provider with platform preconditions missing | Return `platform_preconditions_missing` or equivalent blocked category. |
| Sandbox provider | Return `safe_to_continue_sandbox_only`; do not enable real behavior. |
| Fake provider | Return `safe_to_continue_fake_only`; do not enable real behavior. |
| Unsupported provider | Return `unsupported_provider`. |
| Sandbox fallback attempt | Return `sandbox_fallback_forbidden`. |
| Publish capability request | Block for real provider unless later accepted batches explicitly allow it; Batch 5 still does not publish. |
| Metrics capability request | Block for real provider unless later accepted batches explicitly allow it; Batch 5 still does not read metrics. |
| OAuth capability request | Block for real provider unless later accepted batches explicitly allow it; Batch 5 still does not implement OAuth. |

## Expected Safe Result Categories

The implementation may use these exact categories or a documented equivalent:

- `real_provider_blocked`: real provider runtime remains blocked.
- `feature_flag_disabled`: feature flag is missing, disabled, malformed, revoked, expired, or policy denied.
- `kill_switch_active`: kill switch is active, missing, malformed, revoked, expired, or policy denied.
- `capability_disabled`: requested capability is not explicitly allowed.
- `platform_preconditions_missing`: environment or platform preconditions are missing, malformed, expired, disabled, or policy denied.
- `unsupported_provider`: provider id is unknown or unsupported.
- `sandbox_fallback_forbidden`: real provider attempted to fallback to sandbox or another provider.
- `operation_not_allowed`: requested capability or operation is unsupported.
- `safe_to_continue_fake_only`: real-provider controls do not block local fake-only behavior.
- `safe_to_continue_sandbox_only`: real-provider controls do not block sandbox-only behavior.

The backend implementation maps these to safe internal decision values such as `blocked_real_provider`, `blocked_feature_flag_disabled`, `blocked_kill_switch_active`, `blocked_capability_disabled`, `blocked_preconditions_missing`, `blocked_unsupported_provider`, `blocked_sandbox_fallback_forbidden`, `blocked_operation_not_allowed`, `allowed_fake_only`, and `allowed_sandbox_only`.

## Capability Set

Batch 5 recognizes these real provider capability labels for decision evaluation only:

- `oauth`
- `token_exchange`
- `credential_storage`
- `publish`
- `publish_status`
- `metrics_read`
- `disconnect`
- `revoke`

Recognizing a capability label does not enable the capability. Batch 5 keeps real provider runtime blocked.

## Redaction Rules

User-facing responses must not display token, access token, refresh token, secret, client secret, credential, authorization code, raw OAuth state, OAuth state value, cookie, session, API key, bearer, raw request, or raw response values.

Logs must not display sensitive material.

Audit records must not display sensitive material.

Tests may use only obvious fake values for negative assertions. Test fixtures must not use values copied from real browser URLs, provider callbacks, token responses, cookies, sessions, platform payloads, local secrets, or environment secrets.

## Dependency Gates

1. Batch 2 before state handling.
2. Batch 3 before token exchange boundary behavior.
3. Batch 4 before credential reference boundary behavior.
4. Batch 5 only feature flag / kill switch foundation.
5. Batch 6 may create local user-confirmed publish intent records, but it still must pass Batch 5 controls and still must not execute real publish, upload, scheduled publish, or metrics reads.
6. Future real OAuth / publish / metrics batches still require separate acceptance.

## Explicit Non-Goals

Batch 5 does not provide:

- Real OAuth.
- OAuth URL creation.
- OAuth start route.
- OAuth callback route.
- Real token exchange.
- Real token storage.
- Real credential storage.
- Frontend OAuth UI.
- External service calls.
- Real Douyin authorization.
- Real publish.
- Real metrics read.
- Upload, publish, or scheduled publishing.
- Treating a Batch 6 local publish intent as a completed provider publication.
- Production, commercial, or SaaS readiness.
