# v1.0 Real Publish Adapter Guarded Contract

This document defines the v1.0 Batch 7 guarded real publish adapter boundary. It is not a public commitment that real Douyin publishing is available.

Batch 7 does not add real OAuth, OAuth URL creation, OAuth start routes, OAuth callback routes, real token exchange, real token storage, real credential storage, real external Douyin calls by default, upload, actual publish, scheduled publish, metrics read, or external service calls.

## Boundary

The guarded adapter may create a metadata-only local publish attempt after a confirmed Batch 6 publish intent exists. The adapter must re-check review readiness, local media readiness, safe metadata, credential reference readiness, provider identity, source type, Batch 5 real provider controls, duplicate attempt prevention, and sandbox fallback prevention.

Creating a guarded publish attempt does not execute a provider publish call. It does not upload media, schedule publishing, call Douyin, store provider responses, or read metrics.

## Input Categories

| Input category | Expected boundary handling |
| --- | --- |
| Valid confirmed publish intent | Create metadata-only guarded publish attempt. |
| Missing publish intent | Return `publish_intent_required`. |
| Cancelled or blocked publish intent | Return `publish_intent_not_ready`. |
| Duplicate publish attempt | Return `duplicate_publish_attempt`. |
| Review item not ready | Return `review_not_ready`. |
| Media artifact missing | Return `media_not_ready`. |
| Metadata invalid | Return `metadata_invalid`. |
| Credential reference missing | Return `credential_reference_missing`. |
| Real provider disabled | Return `real_provider_disabled` or a more specific blocking category. |
| Kill switch active | Return `kill_switch_active`. |
| Platform preconditions missing | Return `platform_preconditions_missing`. |
| Unsupported provider | Return `unsupported_provider`. |
| Sandbox fallback attempt | Return `sandbox_fallback_forbidden`. |
| External publish blocked | Return `external_publish_blocked` or keep external call status blocked / not called. |

## Expected Safe Result Categories

- `publish_attempt_created`
- `publish_blocked`
- `publish_intent_required`
- `publish_intent_not_ready`
- `duplicate_publish_attempt`
- `review_not_ready`
- `media_not_ready`
- `metadata_invalid`
- `credential_reference_missing`
- `real_provider_disabled`
- `kill_switch_active`
- `platform_preconditions_missing`
- `unsupported_provider`
- `sandbox_fallback_forbidden`
- `external_publish_blocked`

Implementation result names may be equivalent, but user-facing and test-visible messages must remain safe and redacted.

## Redaction Rules

User-facing responses must not display token, access token, refresh token, secret, client secret, credential, authorization code, raw OAuth state, OAuth state value, cookie, session, API key, bearer, raw request, raw response, provider response, upload response, publish response, external response, or Douyin response values.

Logs and audit records must not display sensitive material.

Database metadata must not store raw authorization code, raw OAuth state, token, secret, credential, cookie, session, API key, bearer, raw request, raw response, provider response, upload response, publish response, external response, or Douyin response material.

Tests may use only obvious fake values for negative assertions. Test fixtures must not copy values from real provider callbacks, token responses, cookies, sessions, platform payloads, local secrets, or environment secrets.

## Explicit Non-Goals

Batch 7 does not provide:

- Real OAuth.
- OAuth URL creation.
- OAuth start route.
- OAuth callback route.
- Real token exchange.
- Real token storage.
- Real credential storage.
- Real external Douyin call by default.
- Upload.
- Actual publish.
- Scheduled publish.
- Metrics read.
- External service call.
- Production, commercial, or SaaS readiness.

## Future Dependency Gates

Batch 8 defines the local publish status reconciliation foundation with metadata-only reconciliation and status snapshot records, but it still does not query real Douyin status, persist provider / upload / publish / status / external / Douyin responses, or read metrics. Batch 9 defines the local limited metrics read guarded foundation with metadata-only metrics snapshots, freshness visibility, and permission / platform limitation visibility, but it still does not query real Douyin metrics, query real status, persist provider / upload / publish / status / metrics / external / Douyin responses, or enable real metrics read. Real publish execution, real status querying, and real metrics querying still require separate acceptance, platform permissions, credential storage readiness, audit / redaction coverage, and explicit runtime enablement.
