# v1.0 Limited Metrics Read Guarded Contract

This document defines the v1.0 Batch 9 limited metrics read guarded boundary. It is not a public commitment that real Douyin metrics read is available.

Batch 9 creates a local, metadata-only metrics snapshot foundation on top of Batch 8 publish status snapshots. It does not add real OAuth, OAuth URL creation, OAuth start routes, OAuth callback routes, real token exchange, real token storage, real credential storage, real external Douyin metrics calls by default, upload, actual publish, scheduled publish, real status query, or external service calls.

## Boundary

Limited metrics read foundation may create a metadata-only local metrics snapshot only after an existing publish status snapshot is found for the same project. That status snapshot must resolve to an existing publish attempt and publish intent chain.

Creating a metrics snapshot does not execute a provider metrics query. It does not call Douyin, query real status, upload media, publish media, schedule publishing, store provider responses, store status responses, store metrics responses, or store external responses. The only allowed persisted values are safe local metric counts, completion rate in basis points, metrics source, freshness status, timestamps, external query status, safe messages, and status-change reasons.

## Input Categories

| Input category | Expected boundary handling |
| --- | --- |
| Valid publish status snapshot | Create metadata-only metrics snapshot. |
| Missing publish status snapshot | Return `publish_status_required`. |
| Publish status not eligible | Return `publish_not_ready_for_metrics`. |
| Missing publish attempt | Return `publish_status_required` or `publish_not_ready_for_metrics`. |
| Missing publish intent | Return `publish_status_required` or `publish_not_ready_for_metrics`. |
| Real provider disabled | Return `real_provider_disabled` or a more specific blocking category. |
| Kill switch active | Return `kill_switch_active`. |
| Platform preconditions missing | Return `platform_preconditions_missing`. |
| Metrics permission missing | Return `metrics_permission_missing`. |
| Metrics freshness unknown | Return `metrics_freshness_unknown` with visible freshness metadata. |
| Unsupported provider | Return `unsupported_provider`. |
| Sandbox fallback attempt | Return `sandbox_fallback_forbidden`. |
| External metrics query blocked | Return `external_metrics_query_blocked`. |
| Malformed fake metrics fixture | Return `metrics_fixture_invalid`. |

## Expected Safe Result Categories

- `metrics_snapshot_created`
- `metrics_read_blocked`
- `publish_status_required`
- `publish_not_ready_for_metrics`
- `metrics_permission_missing`
- `metrics_freshness_unknown`
- `real_provider_disabled`
- `kill_switch_active`
- `platform_preconditions_missing`
- `unsupported_provider`
- `sandbox_fallback_forbidden`
- `external_metrics_query_blocked`
- `metrics_fixture_invalid`

Implementation names may be equivalent, but user-facing and test-visible messages must remain safe, local, and redacted.

## Redaction Rules

User-facing responses must not display token, secret, credential, authorization code, raw OAuth state, cookie, session, API key, bearer, raw request, raw response, provider response, upload response, publish response, status response, metrics response, external response, or Douyin response values.

Logs must not display sensitive material.

Audit records must not display sensitive material.

Database records must not store raw provider response, raw publish response, raw status response, raw metrics response, raw external response, raw Douyin response, raw request, raw response, token, secret, credential, authorization code, cookie, session, API key, bearer, raw OAuth state, upload response, or publish response material.

Tests may use only obvious fake values for negative assertions. Test fixtures must not copy real provider callbacks, metrics payloads, token responses, cookies, sessions, platform payloads, local secrets, environment secrets, raw request payloads, raw response payloads, or raw Douyin payloads.

## Explicit Non-Goals

Batch 9 does not provide:

- Real OAuth.
- OAuth URL creation.
- OAuth start route.
- OAuth callback route.
- Real token exchange.
- Real token storage.
- Real credential storage.
- Real external Douyin metrics call by default.
- Upload.
- Actual publish.
- Scheduled publish.
- Real status query.
- External service call.
- Production, commercial, or SaaS readiness.

## Future Dependency Gates

Any future real metrics read must be approved separately and must keep provider response redaction, credential readiness, explicit user authorization, platform permissions, app review, auditability, rate-limit handling, freshness visibility, kill switch behavior, and no-fallback source separation outside the Batch 9 local-only foundation. Batch 10 may package user test readiness / release candidate materials, but it must not treat Batch 9 as production or commercial readiness.
