# v0.9 Douyin Provider POC / Sandbox Integration Readiness Checklist

## Purpose

This checklist opens v0.9 Batch 0 as a planning / ADR / checklist-only batch, is extended by v0.9 Batch 1 for backend-only Douyin Provider Adapter Skeleton readiness, is extended by v0.9 Batch 2 for sandbox-only operation simulation readiness, is extended by v0.9 Batch 3 for backend-only provider registry / factory routing readiness, is extended by v0.9 Batch 4 for backend-only sandbox metrics / mock workflow POC readiness, is extended by v0.9 Batch 5 for docs-only roadmap alignment to v1.0, v1.5, and v2.0, is extended by v0.9 Batch 6 for backend-only Douyin sandbox API contract / smoke endpoints, and is extended by v0.9 Batch 7 for a frontend Douyin Sandbox POC Panel. Batch 7 may add sandbox-only frontend UI that calls the Batch 6 sandbox API, but it does not implement real Douyin integration, OAuth, callbacks, OAuth URL generation, OAuth state storage, token exchange, credential storage, real metrics fetching, real upload, real publishing, scheduling, external service calls, database changes, real backend API behavior, real provider behavior, tenant implementation, billing implementation, RBAC implementation, or admin console implementation.

Completing this v0.9 checklist does not mean v1.0, v1.5, or v2.0 is complete. v0.9 POC readiness only prepares later versions.

## Stop Conditions

Stop immediately and open a separate review if any of the following appears:

- Any real token / secret / API key / credential / authorization code / OAuth state appears.
- Any real Douyin API call is added.
- Any OAuth callback route is added without an ADR.
- Any token storage is added without an encrypted credential storage ADR.
- Any upload / publish / scheduling is added.
- Any source separation is broken.
- Any sandbox data is displayed as real data.
- Any unknown provider id falls back to `douyin_sandbox`.
- Any OAuth URL generation is added.
- Any sandbox metrics / mock workflow result is represented as real Douyin account connection, real metrics, real upload, real publish, or real scheduling.
- Any business code outside the approved backend-only adapter skeleton, sandbox-only operation simulation, registry / factory routing foundation, sandbox metrics / mock workflow POC, backend API, database table, frontend UI, real Provider implementation, OAuth implementation, token exchange, or external service call appears in Batch 4.
- Any Batch 5 change adds backend code, frontend code, database schema, migration, API route, real tenant, billing, RBAC, or admin console implementation.
- Any Batch 6 API route performs real provider behavior, reads or writes credentials, writes database state, calls external services, creates OAuth URLs, performs token exchange, fetches real metrics, uploads, publishes, or schedules.
- Any Batch 7 frontend UI provides real OAuth, real provider connect, OAuth URL generation, token viewer, token / secret / credential input, real upload, real publish, real scheduling, external Douyin domain calls, or third-party service calls.
- Any document states that v0.9 POC, v1.0, v1.5, or v2.0 is complete.
- Any document states or implies that current v0.9 can be used for direct-customer production commercialization or customer-of-customer SaaS commercialization.
