# v0.9 Douyin Provider POC / Sandbox Integration Readiness Checklist

## Purpose

This checklist opens v0.9 Batch 0 as a planning / ADR / checklist-only batch, is extended by v0.9 Batch 1 for backend-only Douyin Provider Adapter Skeleton readiness, and is extended by v0.9 Batch 2 for sandbox-only operation simulation readiness. Batch 2 may add deterministic `douyin_sandbox` simulated operation results and tests only; it does not implement real Douyin integration, OAuth, callbacks, token exchange, credential storage, metrics fetching, upload, publishing, scheduling, external service calls, backend APIs, database changes, frontend UI, or real provider behavior.

## Stop Conditions

Stop immediately and open a separate review if any of the following appears:

- Any real token / secret / API key / credential / authorization code / OAuth state appears.
- Any real Douyin API call is added.
- Any OAuth callback route is added without an ADR.
- Any token storage is added without an encrypted credential storage ADR.
- Any upload / publish / scheduling is added.
- Any source separation is broken.
- Any sandbox data is displayed as real data.
- Any business code outside the approved backend-only adapter skeleton and sandbox-only operation simulation, backend API, database table, frontend UI, real Provider implementation, OAuth implementation, token exchange, or external service call appears in Batch 2.
