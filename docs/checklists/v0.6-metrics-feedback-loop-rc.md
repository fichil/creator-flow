# v0.6 Metrics Feedback Loop Release Candidate Checklist

本文档用于 v0.6 local fake/manual metrics workflow 的发布候选验收与 v0.6.0 release decision。v0.6 Batch 4 只做 Metrics Workflow Stabilization / RC checklist 收口；v0.6.0 release finalization 只做 release 文档收口。不新增真实平台能力、不接外部服务、不改变 v0.5 fake publishing workflow、v0.4 scheduled draft workflow 或 v0.3 rendering workflow 语义。

## Scope

v0.6 的当前范围是建立发布后指标反馈的本地 fake workflow 验证路径。当前支持：

- `PublicationRecord` 关联多个 metrics snapshots。
- `PublicationMetricSnapshot` 保存平台无关的基础指标字段。
- `FakeMetricsProvider` 生成 deterministic fake/local metrics。
- Metrics API 支持为指定 `PublicationRecord` 创建 fake snapshot、查询 snapshots list、读取单个 snapshot。
- 项目详情页在 `PublicationRecord` 附近展示 metrics snapshots。
- 用户手动点击 `Generate fake metrics` 创建 fake/local metrics snapshot。
- UI 明确显示 `Fake/local metrics` 与 `Not real platform performance`。
- archived project 允许查看已有 metrics snapshots，但不允许创建新的 fake metrics。

## Verified Workflow

当前 v0.6 本地 fake/manual metrics 主链路：

```text
approved ReviewDraft
-> create PublishIntent
-> confirm PublishIntent
-> create PublicationRecord not_started
-> fake publish
-> PublicationRecord succeeded
-> manually generate fake metrics snapshot
-> view fake/local metrics snapshots on the project detail page
```

验收时应确认：

- 用户完成 v0.5 local fake publishing workflow 后可以得到 `PublicationRecord`。
- 用户可以在项目详情页查看每条 `PublicationRecord`。
- 用户可以手动生成 `fake_local` metrics snapshot。
- 创建成功后只刷新对应 `PublicationRecord` 的 metrics list。
- 多条 metrics snapshots 可以稳定展示。
- 缺失指标以明确 fallback 展示，不伪装成真实平台数据。
- fake/local labels 清晰可见。
- archived project 保持只读，只能查看已有 fake/local metrics snapshots。

## Quality Gates

从仓库根目录执行：

```powershell
cd .\backend
.\.venv\Scripts\python.exe -m pytest

cd ..\frontend
npm.cmd run test -- --run
npm.cmd run build

cd ..
git diff --check
git status --short
```

验收时还应执行安全扫描，确认没有真实密钥、token、API key、secret、私钥、本机绝对路径、SQLite DB、`uploads/`、`node_modules/`、`.venv/`、`dist/`、生成媒体或运行时文件进入 Git。

## Explicit Non-Goals

- 不抓取真实 Douyin 或其他平台指标。
- 不接真实 Douyin API。
- 不实现 OAuth。
- 不保存 access token、refresh token、API key、secret 或平台账号凭据。
- 不调用任何外部服务。
- 不做定时指标同步。
- 不做数据分析推荐算法。
- 不自动优化内容。
- 不新增真实平台 dashboard。
- 不新增图表库或独立 analytics 页面。
- 不把 fake metrics 展示或解释为真实平台表现。

## RC Decision

RC checklist 已通过。v0.6.0 release decision：通过，仅限 local fake/manual metrics workflow。v0.6.0 用于验证 `PublicationRecord` 指标快照的领域模型、fake/local provider 边界、后端 API、项目详情页展示与手动 fake metrics 生成流程。

真实平台指标仍需要后续独立工作，包括用户授权、凭据管理、平台 API 接入、真实 `MetricsProvider`、数据合规边界和质量门禁；这些能力不属于 v0.6.0 release scope。
