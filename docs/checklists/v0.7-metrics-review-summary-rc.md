# v0.7 Metrics Review Summary Release Candidate Checklist

## Scope

v0.7 Release Candidate scope is the local fake/manual metrics review summary workflow. It builds on v0.6.0 `PublicationRecord` metrics snapshots and adds deterministic fake/local review summaries for manual human review.

The RC scope covers:

- `PublicationRecord` can be associated with one or more metrics review summaries.
- Review summaries are generated from fake/local `PublicationMetricSnapshot` records.
- Users can view review summaries on the project detail page.
- Users can manually generate a fake/local metrics review summary.
- Fake/local insight labels are clearly visible.
- Not real platform analysis labels are clearly visible.
- Not real Douyin performance labels are clearly visible.
- The insight is only a manual review reference.
- Missing metrics fields and empty summary text fields have stable fallback display.
- Archived projects remain read-only: existing summaries can be read, but new summaries cannot be created.
- Cross-project `PublicationRecord` and summary access returns 404.

## Verified Workflow

The local fake/manual workflow verified for this RC is:

```text
approved ReviewDraft
-> create PublishIntent
-> confirm PublishIntent
-> create PublicationRecord not_started
-> fake publish
-> PublicationRecord succeeded
-> manually generate fake metrics snapshot
-> manually generate fake/local metrics review summary
-> view fake/local review summaries on the project detail page
```

The workflow must keep `source=fake_local` and `is_fake_local=true` visible and stable. Generating or viewing a summary must not modify `TopicCandidate`, `ScriptDraft`, or `ContentPlan`, and must not trigger upload, publishing, scheduled publishing, or any external service.

## Quality Gates

Run the full local gate from the repository root:

```powershell
cd .\backend
.\.venv\Scripts\python.exe -m pytest

cd ..\frontend
npm.cmd run test -- --run
npm.cmd run build

cd ..
git diff --check
git status --short
```

The security scan must confirm that Git does not include real keys, tokens, API keys, secrets, private keys, credentials, local SQLite databases, uploads, `node_modules`, `.venv`, `dist`, generated media, runtime files, real platform response data, real Douyin credentials, or real OAuth callback credentials.

## Explicit Non-Goals

- 不接真实 Douyin API。
- 不实现 OAuth。
- 不保存 access token、refresh token、API key、secret 或 credential。
- 不抓取真实指标。
- 不做定时同步。
- 不做自动推荐算法。
- 不自动优化内容。
- 不新增真实平台 dashboard。
- 不新增图表库。
- 不新增独立 analytics 页面。
- 不触发真实上传、发布、排期发布或外部服务调用。
- 不把 fake/local insight 展示或解释为真实平台分析、真实平台表现或自动推荐结果。

## RC Decision

v0.7 is an RC candidate after Batch 3 stabilization. The backend and frontend now support the local fake/manual metrics review summary workflow with explicit fake/local boundaries and read-only archived-project behavior.

This is not a v0.7.0 release. Release finalization should only happen after the RC gates pass on the target branch and the fake/local boundaries remain clear in API behavior, UI copy, tests, and documentation.
