# v1.5 Minimum Production Release Readiness Checklist

v1.5 is a future Minimum Production Release target for controlled direct-customer commercial use. It is not the multi-tenant SaaS release and does not default to serving customers' customers.

## Production Deployment Readiness

- [ ] Production deployment guide is complete.
- [ ] Environment configuration is documented without exposing sensitive values.
- [ ] Release rollback plan is documented and tested.
- [ ] Production smoke tests are documented.

## Direct Customer Commercial Readiness

- [ ] Direct customer contract and acceptance boundaries are documented.
- [ ] Managed, single-tenant, limited production, or pilot commercial deployment model is selected.
- [ ] Platform dependency and permission limitations are disclosed.
- [ ] Human-confirmed publishing boundary is documented.

## Customer Acceptance Readiness

- [ ] Acceptance criteria are agreed with the direct customer.
- [ ] User test findings are resolved or explicitly accepted.
- [ ] Support and escalation paths are agreed.
- [ ] Disablement and rollback criteria are accepted.

## Security Review Readiness

- [ ] OAuth, token, provider, publish, and metrics security review is complete.
- [ ] Secrets and credentials are encrypted or otherwise protected by accepted design.
- [ ] Security regression tests pass.
- [ ] Dependency and static security scans pass.

## Privacy / Data Retention Readiness

- [ ] Personal data categories are documented.
- [ ] Data retention policy is documented.
- [ ] Data deletion process is documented.
- [ ] Platform data processing limitations are documented.

## Backup / Restore Readiness

- [ ] Backup schedule is documented.
- [ ] Restore procedure is tested.
- [ ] Credential and provider reference restore behavior is documented.
- [ ] Recovery point and recovery time expectations are documented.

## Monitoring / Alerting Readiness

- [ ] Provider integration monitoring is available.
- [ ] Publish workflow monitoring is available.
- [ ] Metrics sync monitoring is available.
- [ ] Alerts have owners and response expectations.

## Incident Response Readiness

- [ ] Incident response runbook is complete.
- [ ] Credential exposure response is documented.
- [ ] Provider policy or permission incident response is documented.
- [ ] Customer communication workflow is documented.

## Support Workflow Readiness

- [ ] Support intake process is documented.
- [ ] Customer-facing support boundaries are documented.
- [ ] Internal triage and escalation are documented.
- [ ] Support access to customer data is minimized and audited where allowed.

## OAuth / Token / Provider Reliability Readiness

- [ ] OAuth callback, token exchange, refresh, revoke, and disconnect paths are tested.
- [ ] Provider rate limit and error handling are tested.
- [ ] Manual recovery path is documented.
- [ ] Provider health checks are available or intentionally deferred with rationale.

## Publish Workflow Reliability Readiness

- [ ] Publish preflight validation is tested.
- [ ] User confirmation is enforced.
- [ ] Duplicate publish prevention is tested.
- [ ] Publish status reconciliation is tested.

## Metrics Reliability Readiness

- [ ] Metrics permission boundaries are tested.
- [ ] Metrics freshness is visible.
- [ ] Metrics sync failures are observable.
- [ ] Metrics summaries avoid unsupported performance claims.

## Operational Runbook Readiness

- [ ] Deploy, rollback, backup, restore, incident, provider outage, and customer support runbooks exist.
- [ ] Runbook owners are defined.
- [ ] Runbook validation evidence is captured.

## Legal / Commercial Boundary Readiness

- [ ] Direct customer commercial use is explicitly scoped.
- [ ] SLA, support, and availability commitments are documented if offered.
- [ ] Douyin platform policy, review status, API limits, and permission dependencies are disclosed.
- [ ] Customer-of-customer SaaS use is explicitly out of scope for v1.5.

## Explicit v1.5 Boundary

- [ ] v1.5 may target controlled commercial use for direct customers after this checklist is satisfied.
- [ ] v1.5 is not a multi-tenant SaaS release.
- [ ] v1.5 does not default to serving customers' customers.
- [ ] v1.5 does not provide white-label, reseller, or marketplace capability unless a later accepted roadmap item adds it.
