# v0.8 Provider & Credential Security Foundation RC Audit Checklist

## RC Scope

This release-candidate audit covers v0.8 Batch 1 through Batch 15 of the Provider & Credential Security Foundation. The audit is documentation-only and exists to close the security, source-separation, metadata-only, and read-only UI boundaries before any v0.9 Douyin Provider POC / Sandbox Integration work starts.

Current stable release remains `v0.7.0 - local fake/manual metrics review summary workflow`. This checklist does not create a `v0.8.0` tag, does not declare v0.8 released, and does not enter v0.9 POC development.

## Quality Gates

Run the full local gate from the repository root:

```powershell
cd .\backend
.\.venv\Scripts\python.exe -m pytest

cd ..\frontend
npm.cmd run test -- --run
npm.cmd run build

cd ..
git diff --check
git status --short
```

Security scan:

```powershell
git status --short
git diff --name-only
git grep -n -I -E "access_token|refresh_token|token_value|api_key|client_secret|oauth_client_secret|authorization_code|state_value|oauth_state_value|credential_material|encrypted_credential|private_key|BEGIN PRIVATE KEY|raw_request|raw_response|raw_payload|token_expiry_value|token_refresh_response|token_revoke_response|provider_token_response|sqlite|\.db|node_modules|\.venv|dist/|uploads/" -- .
```

The security scan may match documentation boundary language, test deny lists, fake test values, ignore rules, or dependency metadata. RC closure requires confirming that no real token, refresh token, secret, API key, credential, authorization code, OAuth client secret, OAuth state value, SQLite DB, uploads, dist, node_modules, `.venv`, generated media, runtime file, real provider response, real Douyin credential, or real OAuth callback credential is committed.

## RC Audit Decision

RC audit checklist is satisfied when the Batch 16 full backend tests, frontend tests, frontend build, `git diff --check`, `git status --short`, and security scan all pass, and the final report confirms no business code, backend API, DB table, frontend UI, real Provider, OAuth route, token lifecycle workflow, real Douyin integration, `v0.8.0` tag, v0.8 release declaration, v0.9 POC development, or v0.7.0 release scope change was introduced.

This checklist is a release-candidate audit artifact, not a release tag and not production readiness certification.
