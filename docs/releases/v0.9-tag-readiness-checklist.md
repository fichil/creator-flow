# v0.9 Tag Readiness Checklist

This checklist prepares a future tag review. It does not create `v0.9.0`, does not merge `main`, and does not create a GitHub Release.

## Tag Candidate

`v0.9.0`

## Tag Target

The tag target must be confirmed by a human reviewer after merge. Do not create the tag in Batch 9. Do not decide the final tag commit in Batch 9.

## Preconditions

- [ ] v0.9 branch is merged into `main`.
- [ ] Final validation is run on `main`.
- [ ] All tests pass.
- [ ] Smoke checks pass.
- [ ] No secrets are present.
- [ ] Release notes are finalized.
- [ ] Changelog is finalized if the project has a changelog.
- [ ] `v0.8.0` tag is unchanged.
- [ ] `v0.7.0` tag is unchanged.

## Draft Commands Only

Draft only. Do not run in Batch 9.

```powershell
git switch main
git pull --ff-only origin main
git tag -a v0.9.0 -m "v0.9.0 Douyin Provider POC / Sandbox Integration"
git push origin v0.9.0
```

## Post-Tag Review

- [ ] Create a GitHub Release only after human approval.
- [ ] Attach final release notes.
- [ ] Verify public README wording.
- [ ] Verify no production or commercial readiness claim is added.
