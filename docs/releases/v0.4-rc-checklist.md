# v0.4 Scheduled Draft Generation Release Candidate Checklist

本文档用于 v0.4 local fake/manual Scheduled Draft Generation workflow 的发布候选验收。v0.4 Batch 8 已完成 Release Candidate stabilization 与 checklist 收口；当前只覆盖 ContentPlan、GenerationSchedule、fake manual GenerationRun、Review Draft placeholder 和项目详情页 UI，不执行 scheduled GenerationRun，不实现 Scheduler / Trigger Engine。

## 已完成能力

- ContentPlan backend domain foundation：项目级 `content_plans` 表、schemas、API routes、create / list / read / update / enable / disable，以及 missing / archived project 边界。
- GenerationSchedule backend domain foundation：绑定现有 `ContentPlan` 的 `generation_schedules` 表、schemas、API routes、create / list / read / update / enable / disable，以及 missing / archived project、missing / cross-project content plan 边界。
- GenerationRun backend-only manual trigger foundation：`generation_runs` 表、schemas、API routes、fake manual create / list / read，以及 missing / archived project、missing / cross-project content plan、missing / cross-project schedule 边界。
- Review Draft backend-only placeholder：`review_drafts` 表、schemas、API routes、manual run 成功后同步创建 1 条 `pending_review` placeholder，并支持 list / read / approve / reject。
- Project Detail 页 ContentPlan / GenerationSchedule / Manual GenerationRun UI：展示和创建 ContentPlan、启停 ContentPlan、展示和创建 GenerationSchedule、启停 GenerationSchedule、手动触发 fake GenerationRun。
- Project Detail 页 Review Draft UI：展示 Review Draft placeholder、审核状态、草稿摘要、输入来源、热点来源 fallback、GenerationRun / GenerationSchedule 信息，并支持 approve / reject 状态变更。
- archived project read-only 行为：允许读取已有 ContentPlan、GenerationSchedule、GenerationRun 和 Review Draft，但禁止 create / update / enable / disable / trigger / approve / reject。
- manual GenerationRun 成功后刷新 GenerationRuns list 与 Review Drafts list。
- v0.4 frontend component extraction / stabilization：项目详情页中的 v0.4 planning 与 review draft 组件已拆分到 `frontend/src/pages/project-detail/`，行为和 API 调用保持不变。

## 验收清单

- `.\scripts\test-backend.ps1` 或 `backend/.venv/Scripts/python.exe -m pytest` 通过。
- frontend tests 通过。
- `.\scripts\build-frontend.ps1` 或 `npm.cmd run build` 通过。
- security scan 未发现真实密钥、本机绝对路径、运行时数据库、上传文件、构建产物、`node_modules`、`.venv`、runtime preview artifact、生成媒体或私有数据进入 Git。
- ContentPlan backend create / list / read / update / enable / disable 可用，`target_frequency_per_week` 保持 1 到 14 的限制。
- GenerationSchedule backend create / list / read / update / enable / disable 可用；省略 `frequency_per_week` 时沿用 ContentPlan 的目标频率。
- fake manual GenerationRun create / list / read 可用；POST 只允许 manual trigger。
- Review Draft placeholder create / list / read / approve / reject 可用；manual GenerationRun 成功后创建 1 条 `pending_review` placeholder。
- Project Detail 页展示 ContentPlan、GenerationSchedule、GenerationRun 和 Review Draft 信息。
- archived project 可以读取已有 v0.4 记录，但不允许 create / update / enable / disable / trigger / approve / reject。
- manual GenerationRun 成功后，Project Detail 页刷新 GenerationRuns 与 Review Drafts。
- Review Draft 的 `hotspot_source_summary` 为空时显示明确 fallback。
- approve / reject 只改变 `review_status`，不创建 render jobs、subtitle drafts、上传文件、媒体文件或发布记录。
- 当前不会执行 scheduled GenerationRun。
- 当前不会注册或运行 Scheduler / Trigger Engine、后台 worker、APScheduler 或 cron。
- 当前不会创建真实 Topic Candidate、Script Draft、Storyboard、Render Job、Subtitle Draft 或媒体文件。
- 当前不接热点源、Notification Service、真实 AI Provider、`FFmpeg`、TTS、发布或上传能力。
- 当前不读取外部热点、私有账号、ChatGPT 历史、本地文件或浏览器数据。

## 明确不在 v0.4 RC 范围内

- scheduled `GenerationRun`。
- Scheduler / Trigger Engine / 后台 worker / APScheduler / cron。
- 完整 `Review Queue` 页面或完整 Review Queue 业务流程。
- 自动生成真实草稿。
- 真实 Topic Candidate、Script Draft、Storyboard、Render Job、Subtitle Draft 或媒体文件创建。
- 热点源、Notification Service、真实 OpenAI、Claude、Gemini 或其他 AI Provider 接入。
- API key、secret 或 token 保存。
- 真实 `FFmpeg` 渲染、TTS、真实音频、真实字幕文件、真实 MP4 或真实视频播放。
- 发布、排期发布或上传到任何平台。
- v0.3 render/subtitle/preview workflow 修改。
- Docker、CI、生产部署或账号系统。

## Runtime Output 与安全边界

v0.4 RC 的 ContentPlan、GenerationSchedule、GenerationRun 和 Review Draft 当前只写入本地 SQLite 元数据。测试或 smoke 产生的 SQLite 数据库、上传文件、runtime preview manifest、生成媒体或其他运行时输出只能位于 Git 忽略路径中，不能进入 Git。仓库不得提交密钥、token、私有账号信息、本机绝对路径、生成媒体、上传素材或私有数据。
