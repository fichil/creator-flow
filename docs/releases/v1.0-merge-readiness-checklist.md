# v1.0 Merge Readiness Checklist

This checklist prepares v1.0 RC materials for maintainer review.

It does not merge automatically and does not create or push tags.

## Branches

- Target merge branch: to be decided by maintainer.
- Source branch: `feat/v1.0-user-test-readiness-release-candidate`.
- Closure audit branch: `chore/v1.0-rc-closure-audit` after this patch.

## Required Checks

- [ ] Backend tests pass.
- [ ] Frontend tests pass.
- [ ] Frontend build passes.
- [ ] `scripts/validate-v0.9-poc.ps1` passes or warnings are reviewed.
- [ ] `scripts/validate-v1.0-user-test-rc.ps1` passes or warnings are reviewed.
- [ ] Documentation consistency is reviewed.
- [ ] No uncontrolled publish path exists.
- [ ] No real external service call path exists.
- [ ] No sensitive value is persisted, logged, committed, or rendered.
- [ ] No fake or sandbox data is presented as real Douyin data.
- [ ] `douyin_real` remains blocked by default.
- [ ] `douyin_real` cannot fallback to `douyin_sandbox`.
- [ ] `v0.9.0` tag remains unchanged.
- [ ] No `v1.0.0` tag exists before an explicit decision.

## Merge Recommendation

- [ ] Eligible for PR review after checks pass.
- [ ] Do not automatically merge.
- [ ] Do not automatically tag.
- [ ] Do not treat review readiness as production, commercial, or SaaS readiness.
