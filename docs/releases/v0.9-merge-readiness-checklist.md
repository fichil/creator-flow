# v0.9 Merge Readiness Checklist

This checklist prepares the human merge review for v0.9. It does not merge `main`, create a tag, create a GitHub Release, or declare the v0.9 POC complete.

## Branch Readiness

- [ ] Source branch is confirmed.
- [ ] Target branch is confirmed.
- [ ] All v0.9 batch commits are included.
- [ ] `v0.8.0` tag is unchanged.
- [ ] `v0.7.0` tag is unchanged.
- [ ] Working tree is not dirty.
- [ ] No force push is required.

## Test Readiness

- [ ] Backend tests pass.
- [ ] Frontend tests pass.
- [ ] Frontend build passes.
- [ ] `scripts/smoke-api.ps1` passes.
- [ ] `scripts/validate-v0.9-poc.ps1` passes or warnings are reviewed.
- [ ] `git diff --check` passes.

## Security Readiness

- [ ] No secrets are present.
- [ ] No real credentials are present.
- [ ] No OAuth state is present.
- [ ] No token storage is added.
- [ ] No real Douyin response is present.
- [ ] No runtime artifacts are committed.
- [ ] No external calls are introduced.

## Documentation Readiness

- [ ] `README.md` is updated.
- [ ] `README.en.md` is updated.
- [ ] `docs/roadmap.md` is updated.
- [ ] `docs/architecture.md` remains consistent.
- [ ] `docs/product-spec.md` remains consistent.
- [ ] `docs/development.md` is updated.
- [ ] v0.9 checklist is updated.
- [ ] RC checklist is updated.
- [ ] Test matrix is available.
- [ ] Release notes draft is available.
- [ ] PR description draft is available.

## Product Boundary Readiness

- [ ] Scope remains sandbox-only.
- [ ] `douyin_real` remains blocked.
- [ ] Unknown provider ids do not fallback.
- [ ] No production or commercial readiness claim is added.
- [ ] v1.0, v1.5, and v2.0 remain future roadmap targets.

## Human Approval Gate

- [ ] PR review is required.
- [ ] Release owner approval is required.
- [ ] Tag creation approval is required.
- [ ] GitHub Release approval is required.
- [ ] No automatic merge is allowed.
- [ ] No automatic tag is allowed.
- [ ] No automatic release is allowed.
