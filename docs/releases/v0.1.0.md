# v0.1.0 Release Notes Draft

## Release Title

v0.1.0 Local Runnable Skeleton

## 当前能力

- 本地启动 `FastAPI` backend 和 `React` + `Vite` + `Tailwind CSS` frontend。
- 使用本地 `SQLite` 保存 `ContentProject` 与 `UserMaterial` 元数据。
- 创建、更新和归档内容项目。
- 默认隐藏 archived 项目，并支持 `include_archived=true` 查询。
- 显示项目素材数量。
- 显式添加文本、摘要、项目记录和链接素材。
- 显式上传 image 或 screenshot 文件素材，保存到本地 `uploads/`。
- archived 项目保留已有素材，但禁止继续导入新素材。
- 提供 Windows PowerShell 本地开发、测试、构建和 API smoke 脚本。

## 本地启动方式

在仓库根目录打开 Windows PowerShell：

```powershell
.\scripts\dev-backend.ps1
```

另开一个 PowerShell：

```powershell
.\scripts\dev-frontend.ps1
```

默认 frontend 地址：

```text
http://localhost:5173
```

默认 backend 地址：

```text
http://127.0.0.1:8000
```

## 验证命令

```powershell
.\scripts\test-backend.ps1
.\scripts\build-frontend.ps1
```

启动 backend 后运行：

```powershell
.\scripts\smoke-api.ps1
```

## 已知限制

- 未实现 AI Provider。
- 未实现 AI 选题、脚本生成、分镜生成或素材方案生成。
- 未实现定时生成。
- 未实现视频渲染。
- 未实现抖音或其他平台发布。
- 未实现 `Docker`、`Docker Compose` 或 CI。
- 未实现生产部署。
- 未实现用户账号。
- 未实现自动扫描本地文件、浏览器或私人账号。
- 未实现自动抓取链接内容。
- 未实现 OCR 或图片内容分析。

## 数据与隐私边界

- SQLite 数据库位于 `data/local/creator_flow.sqlite3`，不得进入 Git。
- 上传文件位于 `uploads/{project_id}/`，不得进入 Git。
- 生成媒体、日志、临时文件和本地构建产物不得进入 Git。
- 密钥、token、credential、private key 和本地私有配置不得进入 Git。
- 当前版本只处理用户显式导入的素材，不自动读取私人账号、浏览器会话或本地目录。

## 不包含内容

- AI Provider。
- 定时生成。
- 视频渲染。
- 抖音发布。
- Docker / CI。
- 生产部署。
- 用户账号。
