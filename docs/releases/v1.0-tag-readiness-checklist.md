# v1.0 Tag Readiness Checklist

This checklist is for a future explicit tag decision. It does not create a tag and must not be treated as tag approval by itself.

## Preconditions Before Any Tag

- [ ] RC closure audit is reviewed.
- [ ] Branch lineage is verified from v0.9.0 through v1.0 Batch 10.
- [ ] Backend tests pass.
- [ ] Frontend tests pass.
- [ ] Frontend build passes.
- [ ] `scripts/validate-v0.9-poc.ps1` passes or warnings are reviewed.
- [ ] `scripts/validate-v1.0-user-test-rc.ps1` passes or warnings are reviewed.
- [ ] Security scan is reviewed.
- [ ] Forbidden wording scan is reviewed.
- [ ] No real provider is enabled.
- [ ] No real OAuth is enabled.
- [ ] No real publish, status, or metrics path is enabled.
- [ ] No `v1.0.0` tag exists before an explicit decision.
- [ ] `v0.9.0` tag remains unchanged.

## Allowed Tag Decision

- [ ] Create no tag.
- [ ] Create an RC tag only after explicit maintainer approval.
- [ ] Create a final tag only after a separate release decision.

## Explicitly Forbidden In This Task

- [ ] Do not create `v1.0.0`.
- [ ] Do not create `v1.0.0-rc.1`.
- [ ] Do not modify `v0.9.0`.
- [ ] Do not push tags.
- [ ] Do not declare production readiness.
- [ ] Do not declare commercial readiness.
- [ ] Do not declare SaaS readiness.
