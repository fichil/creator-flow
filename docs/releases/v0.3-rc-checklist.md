# v0.3 Fake Workflow Release Candidate Checklist

本文档用于 v0.3 fake rendering/subtitle/preview workflow 的发布候选验收。v0.3 仍然是 deterministic fake workflow，不生成真实媒体，不接真实 Provider。

## 已完成能力

- FakeRenderer backend domain foundation：render job、render artifact metadata、fake preview manifest metadata。
- Render Jobs frontend UI：展示 render jobs、fake artifact metadata、preview metadata fallback，并触发已有 fake render job API。
- FakeSubtitle backend domain foundation：subtitle draft、subtitle cue metadata、fake subtitle create/list/read/select API。
- Subtitle Frontend UI：展示 subtitle draft metadata 和 cues，并触发已有 fake subtitle create/select API。
- Fake preview manifest backend placeholder：只写入 Git 忽略路径下的 JSON manifest，不生成真实 MP4、音频或字幕文件。
- Fake preview manifest frontend display：只展示 backend 返回的 metadata，不读取本地 manifest 文件，不新增真实 video player。

## 验收清单

- `.\scripts\test-backend.ps1` 通过。
- frontend tests 通过。
- `.\scripts\build-frontend.ps1` 通过。
- security scan 未发现真实密钥、本机路径、运行时数据库、上传文件、构建产物或 runtime preview artifact 被 Git 跟踪。
- `git status --ignored` 中如出现 `data/local/render_previews/` 产物，只能是 ignored runtime output。
- `git ls-files` 不得命中 `data/local/render_previews/` 下的运行时产物。
- archived 项目可以读取 render jobs、subtitle drafts 和 preview metadata，但不允许 create/select。
- 无 selected Storyboard 时，fake render job 和 fake subtitle draft 创建入口应被阻止或返回清晰错误。
- render job list/read API 返回 fake preview manifest metadata，包括 manifest path、MIME type、file size、checksum、fake video metadata 和可选 selected subtitle draft id。
- subtitle draft read/list UI 可以展示 draft metadata、subtitle cues，以及空 cues fallback。
- Project Detail 页不要把 fake preview manifest 表达为真实可播放视频。
- 前端不得访问或读取 `data/local/render_previews/` 中的 manifest 文件内容。
- 不得新增 `<video>` 真实播放器行为。

## 明确不在 v0.3 范围内

- 真实 `FFmpeg` 渲染。
- TTS 或真实音频生成。
- `.srt`、`.vtt` 或其他真实字幕文件生成。
- 真实 MP4 生成或播放。
- 发布、排期发布或上传到任何平台。
- 真实 OpenAI、Claude、Gemini 或其他 AI Provider 接入。
- API key、secret 或 token 保存。
- `ContentProject.status` 修改。
- Docker 或 CI。

## Runtime Output 边界

fake preview manifest 的 runtime 路径位于 `data/local/render_previews/`。仓库通过 `.gitignore` 中的 `data/local/` 忽略该目录；测试或本地运行产生的 manifest 只能作为 ignored runtime output 存在，不能进入 Git。
